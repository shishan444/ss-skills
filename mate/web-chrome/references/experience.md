# 站点经验学习：读前 / 写后协议 + 拦截记录

> 加载时机：访问新站点前 + 操作成功/被拦截后（SKILL.md 端到端流程的"经验"阶段）

## 核心理念

每访问一个新站点，你都在积累经验：这个站点的脾气、什么操作有效、什么会触发警报。**被拦截不是失败，而是最珍贵的学习时刻**——它告诉你站点的防御边界在哪。记录下来，下次绕开。

经验按域名存储在 `site-patterns/{domain}.md`，跨会话持久。经验内容标注日期，当作"可能有效的提示"而非"保证正确的事实"——站点会变，经验会过期，失败时回退通用模式并更新文件。

## 读前协议（访问站点前）

确定目标网站后，**必须先查经验**：

1. 从 URL 提取域名（如 `https://www.xiaohongshu.com/explore` → `xiaohongshu.com`）
2. 检查 `site-patterns/{domain}.md` 是否存在
3. 若存在，**读取后开始操作**——获取先验知识（平台特征、有效模式、已知陷阱、反爬规避）
4. 若不存在，按通用模式操作，操作成功后创建经验文件

> 经验是"提示"不是"保证"。若按经验操作失败，回退通用模式并**更新**经验文件（标注本次失效）。

## 写后协议（操作成功后）

CDP 操作成功完成后，发现以下任一情况时**主动写入**经验文件：

- 新站点首次成功访问（创建文件）
- 新的 URL 模式 / 选择器 / 操作策略被验证有效
- 平台特征有新发现（内容加载方式、反爬强度变化）
- 已知陷阱有新案例

**只写经过验证的事实，不写未确认的猜测。** 推测用"疑似""可能"标注。

## 写后协议（被拦截时）—— 强制流程

`intercept_detector.py` 检测到拦截信号时，**必须执行**（第一档规则6）：

1. **停止当前操作**，不要反复重试同一方式
2. **记录拦截快照**：URL、症状、触发前最后操作、页面标题/关键词、HTTP 状态
3. **写入站点经验**：在 `site-patterns/{domain}.md` 的"反爬规避"区追加（标注日期）
4. **提醒经验积累**：在回复中明确告知"已记录拦截信息到经验库"
5. **降级策略**：① 拟人 GUI（human_behavior）② 换 locale/UA/headers ③ 告知用户需手动登录

## 经验文件格式

```markdown
---
domain: xiaohongshu.com
aliases: [小红书, RED]
updated: 2026-07-06
---
## 平台特征
架构、反爬强度、登录需求、内容加载方式等事实。

## 有效模式
已验证的 URL 模式、选择器、操作策略。
- URL 模式：`https://www.xiaohongshu.com/explore/{noteId}`（2026-07-06 验证）
- 内容选择器：`.note-content`（2026-07-06 验证）

## 已知陷阱
什么会失败及为什么（标注日期）。
- 频繁 `/new` 开页（>5/分钟）触发风控（2026-07-06）

## 反爬规避
【web-chrome 核心经验区——拦截发生时重点积累】

### 触发条件（什么操作/频率触发了拦截）
- 2026-07-06：连续程序化 eval 提取 8 次后，页面跳转验证码页（CAPTCHA 信号）
- 2026-07-06：未滚动直接取 DOM，body 长度 < 200 触发 CONTENT_GONE

### 规避策略（已验证有效的绕过方式）
- 拟人节奏：HumanBehavior.click_selector + human_scroll，每操作间泊松停顿（2026-07-06 有效）
- 必须登录态：用户 Chrome 需已登录小红书（2026-07-06 验证）

### 失败教训（尝试过但无效，避免重复踩坑）
- 设置 UA 为 mobile 无效，仍被拦截（2026-07-06）
- 单纯加 Referer header 无效（2026-07-06）
```

## 拦截记录的要点

写"反爬规避"区时，每条记录必须包含：

| 要素 | 说明 | 示例 |
|------|------|------|
| **日期** | 发现/验证时间 | 2026-07-06 |
| **触发条件** | 什么操作/频率/特征触发了拦截 | 连续 eval 提取 8 次 |
| **症状** | 拦截的表现（对应 intercept_detector 的信号类型） | 跳转验证码页（CAPTCHA） |
| **规避策略** | 已验证有效的绕过方式（或标注"未找到有效规避"） | 拟人节奏 + 登录态 |
| **结果** | 规避策略是否成功 | 有效 / 无效 |

**失败教训同样重要**——记录"尝试过但无效的方式"能避免下次重复踩坑。

## 经验的时效性

站点会更新反爬策略，经验会过期。处理原则：

- 经验内容标注日期，**最近的记录优先采信**
- 若按 3 个月以上的旧经验操作失败，**先更新经验再继续**，不要盲目遵循
- 同一站点多次失败时，考虑经验已全面失效，回退通用模式重新积累
