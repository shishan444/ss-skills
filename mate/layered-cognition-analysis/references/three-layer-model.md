# 三层认知模型参考

## 核心定义

三层递进关系：

```text
事实层 → 关系因果层 → 格局演化层
```

认知深度不取决于掌握多少信息，而取决于能否从事件呈现，穿透到生成事件的力量，再看见支配这些力量的格局与规律。

## 事实层

认识事情本身。边界是具体事件。

关注：

- 发生了什么。
- 事实是否完整准确。
- 重点和责任在哪里。
- 存在哪些现实约束。
- 在事情内部怎样处理。

目标：优化这一次事情的处理结果。

## 关系因果层

认识决定事情如何呈现的力量。不是普通 A 导致 B，而是主体所处的位置、压力、诉求如何改变事件性质、优先级和资源分配。

生成链：

```text
位置与责任
→ 压力与利益
→ 关注点和诉求
→ 对事件的解释与定性
→ 决策和资源分配
→ 事件最终呈现状态
```

目标：理解并影响事情与决定者之间的关系。

分界标准：如果更换具体人物，事情可能明显改变，主要属于关系因果层。

### 做实标准

关系因果层保留参与者、位置、压力、利益、定性权、资源分配这个核心，但不能停在概念罗列。必须把相关因素组织成能解释事实走向的结构。

执行顺序：

```text
展开因素
→ 筛选关键因素
→ 合并同源因素
→ 熔炼因素结构
→ 推出决策含义
```

展开时至少检查：

- 参与者：谁在场，谁缺席但有影响。
- 位置：谁是规则制定者、资源方、执行者、受影响者、旁观者。
- 压力：谁承担成本、风险、指标、责任、舆论、机会损失。
- 利益：谁获得收益、控制权、解释权、安全感、流量、利润或位置提升。
- 定性权：谁能定义这件事是什么、是否重要、是否值得投入资源。
- 资源分配：谁能决定钱、人、时间、注意力、渠道、工具、组织支持。

筛选标准：

- 这个因素是否会改变事实的发展方向。
- 去掉这个因素后，结论是否明显改变。
- 它是否影响定性权、资源配置、责任归属或行动优先级。
- 它是否能解释多个事实，而不只是解释一个局部现象。

合并标准：

- 把同一压力来源合并。
- 把同一资源约束合并。
- 把同一参与者位置产生的多个表现合并。
- 把同一因果链上的上游因素和下游表现区分清楚。

熔炼结果必须用一句话或一张小表说明：

```text
这件事真正的因果结构是：[哪些参与者在什么位置上，因为什么压力和利益，通过什么定性权和资源配置，把事实推向什么方向]。
```

不合格输出：

- 只列“用户、平台、创作者、市场”。
- 只写“因为竞争激烈、趋势变化、用户需求”。
- 只说“要平衡利益”，没有说明谁的利益、谁能定性、资源如何流动。

## 格局演化层

认识产生这些力量的地图、位置、规律与方向。

必须包含：

- 识局：看清边界、参与者、资源分布、规则、依赖和主要矛盾。
- 定位：明确用户位置、资源、能力、约束、目的和可承担代价。
- 明道：识别不因个人意愿轻易改变的稳定机制。
- 察势：判断当前条件下的方向性合力。
- 择路：顺势、借势、避势、破局、造势。

目标：优化用户在整体演化中的位置、方向和未来选择空间。

分界标准：如果不断更换人物，相似行为和结果仍反复出现，背后可能存在格局演化层规律。

### 做实标准

格局演化层必须先构建局势，再给策略。没有局势构建，就不能直接说顺势、借势、避势、破局或造势。

局势构建至少包含：

- 边界：这件事属于哪张局，不属于哪张局。
- 参与者：至少两个以上参与方，以及用户在其中的位置。
- 资源分布：钱、权、流量、信任、技术、信息、时间、组织能力怎样分布。
- 规则：显性规则、隐性规则、平台规则、制度约束、文化惯性或路径依赖。
- 主要矛盾：当前最能推动冲突和变化的矛盾是什么。
- 演化方向：哪些力量增强，哪些力量衰弱，时间对谁有利。
- 用户位置：用户能影响什么，不能影响什么，有什么筹码和代价。

格局层要回答：

```text
在这张局里，用户不是单纯“解决问题”，而是在 [位置/资源/规则/趋势] 的限制下，选择 [顺势/借势/避势/破局/造势] 中最合适的一条路。
```

### 形态变更

格局演化层必须回头说明低层形态如何变化：

- 事实形态：事实从孤立事件，变成局势中的症状、节点、信号、约束或机会。
- 因果形态：因果从单线原因，变成多方位置、压力、利益、定性权、资源分配形成的结构。
- 决策形态：行动从处理眼前问题，变成改变位置、利用规则、积累资源、避开劣势或创造新局。

## 升维回校机制

三层不是单向上升，而是“上升理解 → 回落校准”的循环。

低维信息支撑升维；高维意图一旦被识别，会自然修订低维焦点：

- 关系因果层会重排事实层里哪些事实重要。
- 格局演化层会重排关系因果层里谁重要、什么关系重要。
- 最终问题定义可能从“怎么处理这件事”改为“怎么改变定性权、位置、势能或未来选择空间”。

每进入高一层，都要回头问：

- 低层关注的重点是否仍然正确。
- 哪些原本重要的点变次要。
- 哪些被忽略的点变关键。
- 原来的方案是否只是低层局部最优。

## 边界

- 更高层不等于更正确。事实不完整时，宏观分析会建立在错误地基上。
- 原因分析不一定属于关系因果层。只有进入位置、压力、诉求、定性权和资源分配，才进入第二层。
- 长期思考不等于格局演化。第三层必须呈现局面、位置、规律和演化方向。
- “大道至简”必须来自复杂规律经过理解和验证后的压缩，不能省略证据。
- “分析上真正做到”必须能展示因素结构、局势构建和形态变更；只写概念名词不算完成。
