# 输出模板

用于需要完整交付时。可以按任务复杂度压缩，但不能丢失升维回校。

选择完整三层时，必须保留以下一级标题：

- `## 重新定义`
- `## 三层分析`
- `## 关系因果结构熔炼`
- `## 格局构建`
- `## 升维回校`
- `## 原问题重写`
- `## 最关键决定变量`
- `## 推荐方向`
- `## 第一步行动`
- `## 验证信号`
- `## 止损线或反转条件`
- `## 仍需补充的信息`

不得用自由段落替代 `关系因果结构熔炼`、`格局构建`、`升维回校` 三个表格。

```markdown
## 重新定义

[用一段话说明：这件事不只是 X，更准确地说是 Y。]

## 三层分析

| 层级 | 已确认信息 | 关键缺口 | 核心判断 | 对应行动 |
| --- | --- | --- | --- | --- |

完整三层分析不得删除“决策含义”列。

| 因素组 | 参与者/位置 | 压力与利益 | 定性权/资源 | 会推动的事实方向 | 决策含义 |

|  |  |  |  |  |  |
```

## 压缩输出

当用户只是临时讨论，不需要完整报告时，用压缩版：

```markdown
我认为这件事要先改定义：[新定义]。

事实层看到的是：[事实层判断]。
关系因果层看到的是：[关系因果判断]；熔炼后的因素结构是：[结构]；它把事实层焦点从 [A] 改成 [B]。
格局演化层看到的是：[格局判断]；真实局势是：[局势]；它进一步把决策方向从 [C] 改成 [D]。

所以原问题应改写为：[新问题]。
当前最优方向：[方向]。
第一步：[行动]。
需要验证：[关键未知变量]。
```

## 边界校准输出

当用户只要求复述意图、安装工具、修改触发条件或确认概念边界时，不要强行展开完整表格。使用：

```markdown
这件事不适合展开成长篇三层分析，它更像是：[任务类型]。

事实层：用户明确要做的是 [事实]。
关系因果层：真正要避免/影响的是 [关系、触发权、定性权或资源配置]。
格局演化层：它反映的长期规则是 [规则/治理原则]。

因此原问题应从 [表面问题] 改写为 [更准确的问题]。
最小动作：[动作]。
验证方式：[验证信号]。
```

### 复述型边界校准

当用户明确要求“理解我的意图，进行复述”时，用更窄的格式：

```markdown
你的意图不是 [容易误解的方向]，而是 [准确意图]。

事实层：你明确指出了 [用户明说内容]。
关系因果层：你真正想修订的是 [定性权/方法边界/资源关系/触发权]。
格局演化层：它背后的长期原则是 [原则]。

因此后续应该把问题从 [旧问法] 改成 [新问法]。

边界提醒：[若继续推进，下一步才需要验证什么；本轮不展开方案。]
```

### 压缩输入不确定性提示

当输入很短时，在正文前加：

```markdown
这段输入信息很压缩。以下区分三类：用户明说的是 X；我为了继续分析暂时推断的是 Y；真正需要验证的是 Z。
```
