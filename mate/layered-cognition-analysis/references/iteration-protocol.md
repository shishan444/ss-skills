# 迭代验证协议

用于用户要求验证、优化或迭代 `layered-cognition-analysis` 时。目标不是证明 skill 已经有效，而是用独立样本暴露失败模式，再把失败模式写回 skill。

## 验证循环

每轮迭代按以下顺序执行：

```text
选择测试 prompt
→ 用当前 skill 独立输出
→ 按 evals/rubric.md 评分
→ 记录失败模式
→ 判断失败属于 SKILL.md、参考文件、输出模板还是测试样本问题
→ 修订最小必要文件
→ 用未见过或未泄露答案的 prompt 再测
→ 对比改进是否真实发生
```

## Subagent 前向验证

可以使用 subagent，但要保持验证完整性：

- 把 subagent 当作真实用户请求执行者，不要告诉它“你正在测试某个失败点”。
- 传入 skill 路径和原始 prompt，不传入预期答案、修订意图或评分结论。
- 每轮并发不超过 2 个 subagent。
- 同一轮至少覆盖两个不同领域；不要只用一个熟悉场景证明有效。
- 评分时以输出为准，不以 subagent 的自我评价为准。

推荐 subagent 提示：

```text
Use $layered-cognition-analysis at /path/to/layered-cognition-analysis to analyze:
[原始 prompt]
```

## 多领域测试分组

至少覆盖这些范式：

1. 自媒体/内容创作：验证注意力、平台、用户参与、账号建立。
2. AI 产品/金融工具：验证策略设计、收益承诺、风险与普通用户位置。
3. 人生/迁移决策：验证城市、行业、出国、岗位留下来的选择空间。
4. 社会形势/压缩输入：验证信息不完整时能否追问或做条件化分析。
5. 普通人 AI 积累：验证资源约束、能力积累、外部势能与可操作路径。
6. 情绪与关系能力：验证感性/理性转换、共鸣练习和关系反馈机制。

## 失败模式归因

发现低分时，先归因再修改：

| 失败模式 | 表现 | 应改位置 |
| --- | --- | --- |
| 事实层摘要化 | 只复述题目，没有区分事实、推断、缺口 | SKILL.md 事实层、question-bank |
| 因果层散点化 | 只列角色/利益，没有合并成因素结构 | three-layer-model、output-template |
| 因果层泛原因化 | 写技术、趋势、心理等普通原因，没有进入位置、压力、利益、定性权、资源 | SKILL.md 关系因果层 |
| 格局层宏大化 | 只讲时代、趋势、社会变化，没有局势边界和参与者 | three-layer-model、rubric |
| 策略动作空转 | 直接给顺势/破局，但没有用户位置、资源和约束 | SKILL.md 格局演化层 |
| 回校缺失 | 三层并列陈述，没有改变低层定义 | output-template |
| 建议不可验证 | 给方向不给第一步、验证信号、止损线 | SKILL.md 最优方向 |

## 通过标准

- 单轮平均分达到 85/100 以上，才算当前轮通过。
- “因素结构熔炼”低于 15/20，必须继续迭代。
- “局势构建与演化”低于 15/20，必须继续迭代。
- “高维回校与形态变更”低于 12/15，必须继续迭代。
- 连续两轮使用不同领域 prompt 达标，才可以认为改进具有泛化性。
