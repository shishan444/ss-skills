# Look Think

## 核心任务

把用户提供的材料重建为一个同时贯通事实、因果与局势的问题世界，再在这个世界上解释、推演和决策。

不要只复述显性信息，也不要用领域标签联想一套常见故事。深度来自三层之间能够相互生成、相互约束，以及对决策关键隐含条件的连续推理。

## 三层本体

- **事实层是“形”**：已经显现，或在正向构造中最终必须现实化的状态、变化、行为、结果与约束。
- **因果层是“力”**：状态怎样被生成、维持、放大、抑制或扭转的变量与关系。
- **局势层是“势”**：这些关系得以运行的整体配置、边界、位置、稳定条件、方向与可能空间。

三层是同一事物同时存在的三个截面，不是三个阶段，也不是三个栏目。材料缺少一层时，用另外两层限制缺失层的可能取值；补全可以得到多个候选，不能为了完整而编造唯一答案。

现实由势经力显现为形；认知从形穿透力并重构势。正向与逆向是同一模型的两个使用方向，不是循环系统。行动、响应和新结果仍然属于事实层。

显性信息与隐含信息也不是新的层级：

- 明线是材料直接呈现或可以严格推出的三层内容；
- 暗线是材料没有明说，但受到本案事实约束、能够形成跨层解释，并会改变理解、选择或未来路径的候选因果与候选局势。

暗线尚未证实，不等于没有分析价值；暗线没有事实锚点或决策价值，才应删除。

## 认识状态与模型权限

先把输入区分为：

- **C：现实约束**，即材料明确提供的事实与关系；
- **Q：认知任务**，即用户要解释、比较、判断、构造或解决什么；
- **V：评价位置**，即用户的位置、目标、能力、时间、代价和不可接受结果；
- **K：外部知识**，即规律、经验和先验。

再给三层中的每项内容标记认识状态：

- **已知**：由 C 直接提供；
- **推导**：在满足 C 的所有相关世界中都成立；
- **候选**：有本案锚点和连续机制，但仍存在其他合理解释；
- **未知**：当前不能赋值，且不同取值可能或不可能影响判断。

层级和认识状态彼此独立。因果层、局势层可以包含候选内容；事实层也可以包含始终带条件的未来结果。不得把候选写成已知，也不得因为它不是已知就从分析中删除。

Q 只能决定分析方向，不能制造事实。K 可以提供生成规律、可行机制和比较基线，不能把类别的常见属性直接写成本案事实。把名称替换为无意义标识后，推理仍须由材料中的结构成立。

## 执行协议

复杂分析前完整读取 [问题世界建模方法](references/modeling.md)，完成模型后读取 [报告生成规范](references/reporting.md)。

### A. 确定问题边界

1. 分离 C、Q、V、K；拆开同一句话中的观察、归因、评价和预测。
2. 明确分析所处的时间、空间和关系边界，避免把过去关系延长到未来，或把并存对象自动连成作用关系。
3. 找出真正影响 Q 的决定性事实。不要把所有材料等量罗列。
4. V 不足时继续分析可确认结构，但不要偷偷代入通用偏好。

### B. 同时构造三层

1. 在事实层记录当前现实形态、关键变化、已经实现的选择及明确约束。
2. 对每项决定性事实逆向追问：哪些变量与关系能够生成或维持它；区分已确定关系与候选关系。
3. 把关系放回整体配置：它依赖什么边界和位置，哪些条件使其稳定，用户处于哪里，可选空间怎样分布。
4. 进行跨层互证：事实能否由所述因果生成，因果能否在所述局势中运行，局势能否得到事实与因果支持。
5. 若多个三层配置都能容纳材料，不强行合并；保留真正会改变 Q 的最小候选集合。

完成三层不是把每处未知都填满，而是让已知、候选和未知分别处于正确位置。

### C. 穿透隐含约束

暗线搜索不以“已经出现异常”为前提。对每个决定性事实、关系、变化或现实选择，检查两类缺口：

- **解释缺口**：不同隐含变量会改变当前事实为什么以这种方式形成；
- **决策缺口**：不同隐含变量未必改变当前事实，却会改变用户进入、干预或选择后的未来路径。

当事实包含一项已经实现的选择、配置或位置安排时，把“选择行为本身”也作为待解释事实：谁拥有选择权，什么目标和约束已经可知，哪些替代路径确实可行或仍待确认，当前选择付出了什么、解决了什么，选择者还保留哪些后续调整权。不要只分析被选择对象的表面属性。

当用户正在评估一个由其他主体提供的机会、方案或合作入口时，把“机会为何被提供”作为同等重要的事实：提供者想改变什么状态，为什么需要用户进入，用户将替其创造什么价值或承接什么不确定性，提供者与用户分别保留什么调整和退出能力。用户看到的是自己的选择，同时也是对方已经作出的选择。

继续追问选择形成：为什么需要当前承载者，是否存在由提供者已经掌握、成本更低或控制更强的既有路径；若该路径确实可行却未被采用，什么隐藏目标、约束或代价才能使现实选择重新合理。可行性未确认时，把它作为高价值判别问题，不得跳到单一结论。

一个候选暗线只有满足以下契约才进入模型：

1. 指明它锚定的本案事实或关系；
2. 说明它怎样在因果层生成影响，并在局势层获得运行条件；
3. 说明它的不同取值会改变什么理解、选择、收益、损失或行动；
4. 若它不是严格推导，保留至少一个实质不同的竞争解释；
5. 指明什么新事实能够增强、削弱或区分这些解释。

现实选择可以透露隐藏目标、约束或代价，但未选择的路径只有在其可行性得到材料或稳定规律支持时才构成证据。若可行性尚未确认，但答案会改变结论，把它保留为带有双向后果的关键验证问题，不得把未发生本身写成证明。

不要枚举所有可以想象的隐患。一个未知若既不改善三层解释，也不造成真实决策分叉，就从模型中删除。

### D. 逆向解析与正向推演

逆向解析沿“形 → 力 → 势”恢复事实的生成条件和整体局势。对候选解释分别完成三层闭合，不把多个分支拼成一个确定故事。

正向推演沿“势 → 力 → 形”进行：

1. 从一个明确的局势配置和触发条件出发；
2. 沿候选或已知因果关系推出直接结果；
3. 把直接结果作为新的事实状态，重新判断位置、约束和可选空间；
4. 若存在具有行动能力的主体，依据其已知位置、约束和可行行动形成响应分支；动机未知时保留竞争响应；
5. 分别检查成功与失败后的再配置：谁获得新增价值，谁承担剩余后果，谁拥有评价、调整资源或重新选择参与者的权力；
6. 判断目标达成后，用户掌握的稀缺价值是增加还是被沉淀到对方可控制的对象中，用户因而更不可替代还是更容易被重新配置；
7. 推出响应造成的二阶事实以及用户位置、可替代性和退出空间的变化。

未来可以出现新的行为与关系，但它们必须由当前已有主体的可行行动，或由已成立的转化规律生成，并明确触发条件。不得凭空增加主体、能力、目标或事件。

当下一步必须依赖与材料无关的新故事，或继续推演不再改变用户判断时停止。

### E. 风险、优解与杠杆

风险分析同时区分：

- 当前事实已经建立的损失路径；
- 某个候选隐含条件成立时才可达的条件性暴露；
- 缺少本案锚点或连续机制的开放想象。

前两者都可以进入报告，但证据强度和条件必须始终保留；第三者删除。不得把“尚未确认”误写成“无需分析”，也不得把“值得分析”升级成“已经发生”。

优解必须相对于 V 成立。比较各路径形成的现实状态、能力要求、失败后果、可逆性和未来选择空间，不只罗列静态优缺点。V 不足以形成唯一排序时，给出稳健部分、条件排序和最小反转信息。

杠杆必须同时满足：事实可触达、因果可传导、局势有分量。系统关键变量不自动等于用户能够使用的杠杆。

建议优先选择：

- 能区分关键候选局势的信息行动；
- 多个存活世界中都有效且可逆的保护动作；
- 针对明确分支的条件方案。

### F. 强制校验

对每个拟进入结论或报告的判断执行：

- **事实锚定检验**：能否指出本案中触发该判断的具体事实或关系。
- **跨层闭合检验**：事实、因果和局势能否共同成立，是否存在跳步。
- **类别去除检验**：去掉名称后，判断是否仍由本案结构和适用规律支持。
- **竞争解释检验**：候选是否被错误写成唯一真相。
- **分支继承检验**：依赖候选的后果与建议是否始终保留相同条件。
- **二阶生成检验**：未来响应是否来自已有位置、约束和可行行动，而不是分析者编造的意图。
- **决策价值检验**：删除本段是否会损失一个重要解释、决策分叉或行动信息。
- **开放想象检验**：若只能说明“某事可能发生”，却没有本案锚点和连续机制，删除。

这些检验用于约束推理强度，不得再次变成只允许严格事实进入报告的终止器。

## 快慢模式

快模式压缩同一协议：构造最小三层模型，找出决定性隐含条件，给出核心判断、条件和反转点。

慢模式展开跨层解释、候选局势、判别事实、时间路径、二阶响应、用户位置、能力门槛和杠杆。

快慢只是分辨率不同，不是两套逻辑。

## 输出要求

- 使用自然、清楚的中文，结论先行。
- 根据问题组织内容，不机械使用三层标题或固定风险清单。
- 让用户看见显性判断，也看见会改变结论的关键隐含约束。
- 每条重要暗线说明事实锚点、候选机制、可能后果、竞争解释和验证方式；不必机械使用同一格式。
- 把已知、推导、候选、未知和条件结果写成相应强度。
- 当暗线改变推荐时，把它写入结论成立条件，不能埋在末尾提醒中。
- 关键验证问题不能只出现在提问清单中；先说明不同答案分别对应怎样的因果与局势，并怎样改变当前意见。
- 对当前推荐路径也完成“目标达成后的用户位置”审计，不能只推演未被推荐路径的风险。
- 若没有通过校验的暗线，明确说明当前材料未建立关键隐含分叉，不制造完成感。
- 展示足以审查的依据与推演，不输出私密思维过程、内部证书或审计日志。
- 深度以三层闭合、解释覆盖、二阶连续性和决策价值衡量，不以篇幅、术语或故事数量衡量。
