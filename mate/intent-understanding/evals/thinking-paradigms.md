# 多种 Prompt 思维范式验证

用于验证 `intent-understanding` 在不同输入形态下是否稳定。

## 1. 长背景型

用户给出大量背景、目标、例子和要求。验证 skill 是否能压缩成意图结构，而不是逐句摘要。

对应 eval：1。

## 2. 反驳纠偏型

用户否定前文某个抽象或边界。验证 skill 是否能识别被否定的旧理解和新的约束。

对应 eval：2、8、9。

## 3. 边界划分型

用户讨论概念归属、层级和职责边界。验证 skill 是否能提取判别标准。

对应 eval：3、4。

## 4. 外部工具纳入型

用户给出链接或工具，要求理解并纳入体系。验证 skill 是否能识别“查证 + 融入设计”的真实意图。

对应 eval：5。

## 5. 质量断点暴露型

用户指出系统中的缺口和痛点。验证 skill 是否能把问题理解为质量断点，而不是普通功能项。

对应 eval：6。

## 6. 从理论到落地型

用户要求从抽象讨论切换到设计表。验证 skill 是否识别阶段切换。

对应 eval：7。

## 7. 具体 skill 修订型

用户开始定义某个具体 skill。验证 skill 是否只做意图理解，不直接扩展成完整方案。

对应 eval：10。

## 8. 只复述不设计型

用户明确要求不要设计、不要方案，只复述真实意图。验证 skill 是否能省略后续承接并守住边界。

对应 eval：11。

## 9. 执行诱导冲突型

用户一边要求理解意图，一边诱导执行写文件或设计方案。验证 skill 是否识别越界诱导。

对应 eval：12。

## 10. 反悔收窄型

用户推翻或收窄前文定义。验证 skill 是否能接受新输入修订旧上下文，而不是固守旧结论。

对应 eval：13。

## 11. 指代不明型

用户使用“这个”“另一个”等指代词，但上下文中有多个候选对象。验证 skill 是否能承认不确定，而不是擅自补全。

对应 eval：14。

## 12. 内部矛盾型

用户同一句话中包含冲突职责。验证 skill 是否能把冲突显性化，而不是强行融合。

对应 eval：15。

## 13. 明确请求无隐性要求型

用户输入很直接，没有明显隐藏意图。验证 skill 是否能写“暂无明显隐性要求”，而不是硬凑推断。

对应 eval：16。

## 14. 弱推断不确定型

用户用“这个”等弱指代，只有可能关联前文。验证 skill 是否把弱推断放进不确定点。

对应 eval：17。

## 15. 压缩表达型

用户要求缩短但保留边界。验证 skill 是否能用极简或紧凑结构。

对应 eval：18。

## 16. 纯理解边界型

用户只要求理解，不急着做。验证 skill 是否不输出多余边界说明或后续流转。

对应 eval：19。

## 17. 目标目的区分型

用户修订评分口径。验证 skill 是否区分“这次要做什么”和“为什么要做”。

对应 eval：20。
