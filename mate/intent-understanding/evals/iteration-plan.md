# intent-understanding 迭代验证循环

目标：验证 `intent-understanding` 是否能在多领域 prompt 中完整呈现真实意图，尤其是事件本身、上下文关系、因果链条、价值指向、显性条件、隐性条件、不确定点和边界约束。

## 1. 测试集

每轮至少跑三组评测：

- 基础评测：`evals/evals.json`，验证上下文关联、修订意图和边界控制。
- 挑战评测：`evals/challenge-evals.json`、`evals/v03-targeted-evals.json`，验证冲突、反悔、弱推断、不越界。
- 复杂意图评测：`evals/complex-intent-evals.json`，验证多领域复杂 prompt 下的因果、价值和条件还原。

## 2. 评分规则

使用 `evals/rubric.md`，每条满分 10 分。

重点观察：

- 真实意图是否完整，不只是“用户想问什么”。
- 是否说明因果链条和用户担心的后果。
- 是否说明用户希望产生的价值或避免的损耗。
- 是否区分显性条件、隐性条件和弱推断。
- 是否守住边界，不直接进入方案设计或执行。

通过线：

- 单条低于 7 分：该类 prompt 未通过。
- 复杂意图评测均分低于 8 分：真实意图还原能力不足。
- 总体均分低于 8.5 分：继续迭代。
- 改动后总分必须严格高于改动前，且不能牺牲边界控制。

## 3. Subagent 验证方式

每轮至少使用 1 个独立 subagent 进行盲评。可用资源允许时，使用 2 个 subagent：

- Judge A：只看 `SKILL.md`、`rubric.md` 和测试 prompt，做 dry-run 评分。
- Judge B：同样只看必要文件，重点找失败模式和退步点。

不要把主流程的诊断、预期改法、目标分数泄露给 subagent。subagent 只收到：

- skill 文件路径。
- rubric 文件路径。
- eval 文件路径或测试 prompt。
- 输出要求：逐条评分、指出短板、给出保留/回退建议。

## 4. 单轮迭代流程

```text
基线评分
→ 找到最低维度或最常见失败模式
→ 每轮只改一个主要问题
→ 更新 SKILL.md 或 evals
→ subagent 复评
→ 对比改前/改后分数和失败样例
→ 保留、继续微调或回退
```

每轮只允许一个主改动方向，避免无法归因。例如：

- Round 1：补强真实意图定义。
- Round 2：补强隐性条件和弱推断边界。
- Round 3：补强复杂 prompt 的输出模板和失败模式。

## 5. 保留和停止规则

保留改动：

- 总分严格高于基线。
- 复杂意图评测均分提升。
- 边界控制不下降。
- subagent 没有指出严重越界或冗余。

回退或重写：

- 总分下降。
- 输出更长但没有更准确。
- 为了因果和价值分析，开始替用户做方案设计。
- 弱推断被写成事实。

停止条件：

- 连续两轮提升都小于 0.3 分。
- 复杂意图评测均分达到 8.5 分以上。
- 新增规则开始让简单 prompt 变得笨重。

## 6. 多领域验证矩阵

| 领域 | 对应 eval | 主要验证点 |
|---|---|---|
| 自媒体/内容 | 21 | 选题、注意力、参与、账号资产之间的价值链 |
| AI 金融产品 | 22 | 策略生成、获利能力、普通用户、风险边界 |
| 人生/职业决策 | 23 | 宏观形势、个人位置、迁移选择、可操作性 |
| 残缺输入 | 24 | 不擅自补全，弱推断进入不确定点 |
| AI 趋势下普通人积累 | 25 | 资源约束、长期积累、避免空转 |
| 情绪沟通训练 | 26 | 感性理性切换、共鸣价值、练习边界 |
