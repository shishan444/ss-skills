# 创作型骨架构建指南

## 识别特征

- 有阶段性（从初始素材到最终产出经历多个阶段）
- 阶段之间存在迭代回环——写完初稿后可能回到前面的阶段修改
- 质量标准比步骤顺序更重要——"做到什么程度"比"先做哪一步"更关键
- 完成标准通常是主观的但有可度量的指标

## 骨架构建方法：阶段划分 + 质量门控 + 迭代回路

### 第一步：识别阶段

从方向声明中提取从初始素材到最终交付经历的关键阶段。每个阶段定义：
- 进入条件：什么状态下开始这个阶段
- 产出物：这个阶段结束时手里多了什么

### 第二步：定义质量门控

为每个阶段之间的衔接点定义质量检查：
- 通过标准：什么状态下可以进入下一阶段
- 回退条件：什么状态下需要回到前面的阶段
- 如果通过标准尚未明确，标注 `[待完善：...]`

### 第三步：标注迭代回路

识别哪些阶段之间存在回环：
- 从哪个阶段回到哪个阶段
- 回退的触发条件是什么
- 回退后从哪里重新开始

### 产出形态

一个带质量门控和迭代回路的阶段图，可能包含 `[待完善：...]` 标注。

## 完整推演示例：公众号文章创作 Skill

### 方向声明

> **触发**：素材已梳理完毕、写作方向已确认，需要创作完整的公众号文章时
> **输入**：结构化素材（主题、核心观点、支撑材料）+ 目标读者画像（可选）
> **输出**：一篇符合公众号排版规范、可直接发布的完整文章
> **约束**：适合手机端阅读；单篇 2000-4000 字

### 骨架产出

```markdown
## 做什么

当素材已梳理完毕、写作方向已确认时，创作一篇符合公众号排版规范、可直接发布的完整文章。
不触发：素材尚未分析、方向未确认的情况。

## 怎么做

### 阶段一：结构设计

1. 根据素材确定文章结构（开头方式、正文分段、结尾收束）
2. 定义每个段落的核心论点和支撑材料
3. 质量门控：结构是否逻辑自洽、支撑材料是否覆盖所有核心论点
   - 通过 → 进入阶段二
   - 不通过 → 回到步骤 1 调整结构

### 阶段二：初稿撰写

1. 按结构设计撰写完整初稿
2. 确保每个段落有明确的论点-论据-论证链
3. 质量门控 `[待完善：定义初稿通过的最低标准——是否要求每段论点清晰、字数是否达标、是否有逻辑断裂]`
   - 通过 → 进入阶段三
   - 不通过 → 回到阶段一步骤 1 重新审视结构

### 阶段三：打磨优化

1. 检查并优化标题（吸引力 + 准确性）
2. 检查并优化开头（是否在 3 句话内抓住读者）
3. 检查段落节奏和信息密度
4. 适配手机端排版 `[待完善：定义手机端排版的具体规范——段落长度、留白、重点标记方式]`
5. 质量门控 `[待完善：定义打磨完成的判定标准——是否需要自检评分达到某阈值]`
   - 通过 → 产出最终文章
   - 不通过 → 回到步骤 1 继续打磨（最多迭代 3 轮）

## 完善指引

1. [高] 阶段二质量门控标准 —— 需要定义初稿"通过"的最低要求（论点清晰度、字数范围、逻辑连贯性），补全后可客观判定是否该回退
2. [高] 阶段三手机端排版规范 —— 需要定义段落长度上限、留白规则、重点标记方式，补全后排版有据可依
3. [中] 阶段三打磨完成的判定标准 —— 需要定义"打磨到位"的可度量指标，可参考 9 维度评分机制
4. [低] 阶段一的结构模板库 —— 可以为常见文章类型（观点型、教程型、案例型）各定义一套结构模板，提升阶段一效率

## 不可破

- 单篇 2000-4000 字
- 必须适配手机端阅读
- 素材未确认方向不触发
```

## 常见留白位置

创作型骨架中最常出现留白的典型位置：

- **质量门控的具体通过标准**：知道要检查，但"通过/不通过"的判定线不够清晰
- **迭代回退的触发条件**：知道可以回退，但什么信号触发回退不够具体
- **素材充分性判断标准**：知道进入阶段一需要"素材已梳理"，但怎么判断素材够不够
- **创作风格/语调定义**：知道要写文章，但文章的语调、风格边界未定义
