# 提取质量标准与判定规则

> `/st-digest` 提取过程中的质量判定依据。

## 核心命题识别

### 什么算"核心命题"
- 去掉它，整个材料的论证链条会断裂
- 它不是常识，也不是纯定义，而是带有判断或机制的陈述
- 通常是作者花最多篇幅论证的东西

### 什么不算"核心命题"
- 纯背景介绍（"历史上曾经..."）
- 术语定义（"X 是指..."）——除非定义本身就是核心贡献
- 举例说明——例子是论证手段，不是命题本身
- 作者的立场声明（"我认为..."）——除非立场本身构成贡献

## 概念提取

### 粒度标准
概念应该是一个**中等粒度**的知识单元：
- **太粗**: "经济学" "机器学习" — 这些是领域，不是概念
- **太细**: "SGD 中学习率在第 50 轮衰减到 0.01" — 这是配置细节，不是概念
- **合适**: "比较优势" "反向传播" "类型推导" — 可以独立理解和迁移的知识单元

### 独立性判定
一个概念应该：
- 能被独立解释（不需要同时解释另一个概念才能理解它）
- 能被独立测试（可以单独出题验证理解）
- 能被独立迁移（可以应用到其他领域）

### 数量控制
- 学术专著: 10-20 个核心概念
- 教科书章节: 5-10 个
- 技术文档/API: 3-8 个
- 论文: 3-6 个

超过这个范围，说明粒度太细，需要合并。低于这个范围，说明粒度太粗，需要拆分。

## 简化层质量标准

### 必须保留的东西
1. **核心机制**：概念如何运作的"因果链"或"结构"
2. **反直觉点**：如果概念的结论和直觉相悖，必须保留（不能为了简化而抹平锐角）
3. **存在理由**：这个概念为什么被提出，它解决了什么矛盾

### 可以省略的东西
1. 严格的数学形式化（移到精度层）
2. 历史背景和学派争论（除非它们是理解核心机制的必要前提）
3. 边界条件和极端情况（移到精度层）

### 禁止的做法
- 不能用另一个未解释的术语来解释当前术语
- 不能把"反直觉点"简化掉——这往往是概念最有价值的部分
- 不能只给定义不给"为什么"

## 精度层质量标准

### 必须回答的问题
1. 简化版具体省略了哪些内容？
2. 这些省略在什么场景下会导致错误判断？
3. 完整的精确表述是什么样的？

### 格式要求
- 必须明确写出"简化版遮掉了：..."，而非含糊地说"还有更多细节"
- 如果简化版没有遮掉重要东西（少数情况），如实说"简化版的精度损失较小，主要省略了..."

## 拓展层质量标准

### 跨域同构的判定标准
有效的同构必须满足：
1. **结构相同**：两个概念的底层运作机制是同一个骨架
2. **可映射**：能明确指出"概念 A 的 X 对应概念 B 的 Y"
3. **非表面相似**：不能只是"都有反馈"这种泛泛之谈

### 有效 vs 无效的同构

**有效**：
- "比较优势（经济学）和 专注优势（管理学）同构：两者都是在问'相对成本/收益的分配'问题"
- "闭包（编程）和 路径依赖（经济学）同构：都是'过去的条件塑造了当前的行为范围'"

**无效**：
- "机器学习和心理学都涉及学习" — 表面词汇相似
- "经济周期和人的情绪波动都是波动的" — 没有结构映射

### 锚点连接
如果学习者 profile 存在，为每个概念尝试找到 1-2 个已有锚点的连接：
- 匹配标准：锚点的结构与新概念的核心机制同构
- 在拓展层末尾标注"与你的已有锚点 '{锚点名}' 存在结构映射"
- 如果找不到合适的锚点，不强行连接

## 召回题目质量标准

### L1（识别）
- 给出足够的信息让理解概念的人能识别，但不能太直接
- 避免使用概念名称本身

### L2（解释）
- 必须是开放式的，不能是选择题
- 必须指定"要求包含 X 要素"，让评估有锚点
- 要素不能太多（2-3 个），否则变成默写

### L3（迁移）
- 情境必须不在原文中出现过
- 情境必须足够新，让学习者需要"迁移"而非"回忆"
- 正确答案不应该是唯一的——只要能识别出同构机制就算通过

## 概念图谱质量标准

### 依赖关系
- 只标注"学 B 之前必须先懂 A"这种硬依赖
- 不标注"了解 A 有助于理解 B"这种软关联
- 必须无环——如果有环，说明概念拆分有问题

### 学习顺序
- 按拓扑排序给出
- 如果多个概念可并行学习，标注出来
