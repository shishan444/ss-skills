# obsidian-forge 设计理由

## 失败模式

**FM1：信息孤岛处理**
AI 只基于笔记原文的几十个字进行全部分析，不搜索外部信息也不深挖背景。产出的"我怎么看"和"关联上下文"都只从原文字面推导，缺乏信息增量。
- 根因：没有被要求在对话过程中主动检索补充信息

**FM2：单轮填表式对话**
AI 按模板字段逐个问一遍就结束，每个字段只问一次。用户给出第一反应后 AI 立刻整理成文，不追问也不挑战。
- 根因：没有设计"追问/质疑"的对话循环机制

**FM3：AI 代思考而非引导思考**
当用户回答模糊或简短时（如"觉得有道理"），AI 为了完成任务会自己补充生成内容。最终笔记中"我怎么看"其实写的是 AI 的观点而非用户的。
- 根因：没有明确"用户没说清楚的不能替用户说"

**FM4：关联浮于表面**
AI 在 vault 中搜索时只用笔记原文关键词做字面匹配，找到的关联是"都提到了同一个词"而非"讨论了相关的逻辑/问题"。网络搜索也只做一次就停止。
- 根因：没有多角度、多轮次的搜索策略

**FM5：有信息无升维/无深钻**
AI 可能检索了相关资料，但只是把信息堆在用户面前，不会做两件事：
- 向上：从具体案例/现象中抽象出可复用的模式、原理、思维框架
- 向下：针对某个关键点深入追究细节，追问"为什么""具体怎样""底层机制是什么"
- 根因：没有区分"信息补充"和"维度提升"是两件不同的事

## 根因分析与推导

### FM1 信息孤岛处理

| 追问 | 判定 | 产出 |
|------|------|------|
| A: 不知道什么是好的？ | 是 | 评价视角：好的笔记处理 = 有外部信息增量的处理 |
| B: 权衡选错优先级？ | 是 | 取舍：效率 vs 深度，应选深度 |
| C: 跳过关键步骤？ | 是 | 缺失：领域知识检索步骤 |
| D: 有隐患没意识到？ | 是 | 代价：无外部信息 = 在原文逻辑里打转 |

### FM2 单轮填表式对话

| 追问 | 判定 | 产出 |
|------|------|------|
| A | 是 | 好的对话 = 多轮追问后用户产生新认知 |
| B | 是 | 取舍：任务完成感 vs 思考深度 |
| C | 是 | 缺失：追问/质疑循环 |
| D | 是 | 代价：用户只说了第一反应，没有触及深层想法 |

### FM3 AI 代思考

| 追问 | 判定 | 产出 |
|------|------|------|
| A | 是 | 好的产出 = 核心观点来自用户，AI 只负责整理和延伸 |
| B | 是 | 取舍：产出完整度 vs 真实性 |
| C | 是 | 缺失：用户模糊时追问而非替代 |
| D | 是 | 代价：笔记写的是 AI 观点，未来回顾时无法复现当初的真实思考 |

### FM4 关联表面化

| 追问 | 判定 | 产出 |
|------|------|------|
| A | 是 | 好关联 = 逻辑/问题层面的连接，非关键词匹配 |
| B | 部分 | 优先找"有意义的"而非"最接近的" |
| C | 是 | 缺失：多角度搜索 + 关联质量评估 |
| D | 是 | 代价：大量弱关联噪音，降低知识网络密度 |

### FM5 有信息无升维/深钻

| 追问 | 判定 | 产出 |
|------|------|------|
| A | 是 | 好的分析 = 不只列信息，要抽象出规律或钻入细节 |
| B | 是 | 取舍：全面展示 vs 维度提升 |
| C | 是 | 缺失：升维分析（向上抽象）和深钻分析（向下追究）两个独立环节 |
| D | 是 | 代价：信息堆砌，用户看了觉得"有用但不知道怎么用" |

## 分歧点清单与强度判定

| 分歧点 | 默认行为 | 应该做 | 根因来源 | 强度 | 判定依据 |
|--------|---------|--------|---------|------|---------|
| DP1 | 不检索外部信息 | 针对主题做领域知识检索（WebSearch + WebFetch + Grep） | FM1 | 强 | 有无此步，产出根本不同 |
| DP2 | 每字段问一次就过 | 关键点追问/质疑/挑战，至少 2 轮 | FM2 | 强 | 单轮 vs 多轮，产出根本不同 |
| DP3 | 用户模糊时 AI 补充 | 追问用户直到说清楚，绝不替用户说 | FM3 | 强 | 决定笔记归属权 |
| DP4 | 关键词字面匹配 | 多角度概念搜索 + 评估关联质量 | FM4 | 中 | 方向性差异 |
| DP5 | 检索后罗列信息 | 升维抽象规律 + 深钻关键细节 | FM5 | 强 | 有无维度提升，产出根本不同 |
| DP6 | 低密度命名（"准备""分析""总结"） | 高密度自带隐喻的命名（"扎根""破壁""淬炼"） | 用户反馈 | 中 | 方向性指导 |

## 设计决策记录

### 流程命名选择：取种 → 定锚 → 扎根 → 破壁 → 淬炼 → 织网 → 结晶

每个名称同时传达动作和隐喻：
- **取种**：起点 + 生长性
- **定锚**：确定核心 + 防止漂移
- **扎根**：向下吸收养分 + 建立根基
- **破壁**：打破认知边界 + 突破
- **淬炼**：反复锻打 + 提炼精华
- **织网**：编织关联 + 知识网络
- **结晶**：液态思考 → 固态笔记

### 与 obsidian-think 的定位区分

| 维度 | obsidian-think | obsidian-forge |
|------|---------------|----------------|
| 核心方法 | 轻量对话引导 | 重度检索 + 深度追问 |
| 信息来源 | 笔记原文 + vault | 原文 + vault + 联网搜索 + 网页抓取 |
| 对话深度 | 每字段一轮 | 关键点至少两轮追问 |
| 分析维度 | 无升维/深钻 | 明确升维抽象 + 深钻追究 |
| 适用场景 | 日常轻量处理 | 需要深度思考的重要碎片 |

### 笔记模板新增字段

在用户原有模板基础上，新增"升维发现"和"深钻理解"两个字段。这两个字段是 DP5 的直接产物，确保维度提升的成果被显性记录，而非消散在对话中。

### 铁律设计

四条铁律分别对应四个强指导点：
1. "绝不替用户说" → DP3（归属权）
2. "先检索再讨论" → DP1（信息增量）
3. "追问至少两轮" → DP2（对话深度）
4. "升维和深钻都要做" → DP5（维度提升）

### 控制属性

未添加任何控制属性（默认）。理由：此 skill 既适合用户主动调用（"锻造笔记"），也适合 Claude 在用户表达深度思考意图时自动推荐。
