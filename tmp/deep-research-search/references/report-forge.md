# 报告锻造

知识图谱和熔炼文件已经完成了"理解"这件事。报告锻造是把理解按受众需要的脉络重新组织成可交付的文字。

---

## 锻造流程

### 第 1 步：读熔炼文件，把握全局

读取 `smelting.md`，获得全局理解：全景判断、跨节点暗线、核心张力、共享沉默、理解地图。

如果某个地方需要回查具体数据或引述，再去图谱节点中翻取。但全局理解本身不需要重新从图谱综合——熔炼文件已经做好了这件事。

### 第 2 步：确定受众 + 领域

从 `research-brief.md` 取出暂存的受众判断和领域归属：

| 受众 | 报告焦点 | 详略策略 |
|------|---------|---------|
| 技术决策者 | 对比数据、选型建议、风险评估 | 技术细节充分展开，商业概述精简 |
| 业务决策者 | 市场格局、趋势判断、投资回报 | 商业分析充分展开，技术原理精简 |
| 学习者 | 概念脉络、入门路径、里程碑 | 基础概念充分展开，前沿议题概述 |
| 不确定 | 面向普通人的通用报告 | 均衡覆盖，避免专业术语堆砌 |

领域决定报告的**思维线索**——用什么逻辑把内容串起来：
- 技术类 → 问题驱动（"什么问题 → 怎么解决 → 效果如何"）
- 商业类 → 价值驱动（"市场多大 → 谁在竞争 → 趋势往哪走"）
- 学术类 → 争议驱动（"核心分歧 → 各方证据 → 当前共识"）
- 社会类 → 影响驱动（"谁受影响 → 怎么影响 → 各方反应"）

**禁止万能模板。** 每个领域的思维线索来自领域本身的内在逻辑，不是套用固定章节。上面的四种只是常见示范，如果领域不属于这四种，从领域自身推导逻辑。

### 第 3 步：撰写报告

**受众焦点处的篇幅充分展开，非焦点处精简概述。**

撰写原则：
- 按领域自带的思维线索组织逻辑
- 每个论点标注数据来源（`[来源: data/round-N/xxx.md]`）
- 按需从图谱节点回查详细数据
- 信息不足的维度诚实标注（"本维度数据有限，建议补充调研"）
- 矛盾发现保留（不同来源的冲突本身有价值，呈现各方观点 + 可信度判断）
- **报告中最有价值的段落不是"我确认了什么"，而是转折——"我之前以为不对的东西实际上有道理"或"看起来确定的事情其实还有争议"。把这些转折写进去，报告才有灵魂。**

### 第 4 步：图谱导出

```bash
python ${CLAUDE_SKILL_DIR}/scripts/graph_manager.py export -g {workdir}/graph
```

输出 `graph/_overview.md`，作为报告附录，让读者看到研究的知识全景。

> 研究周期：{日期}
> 数据源：{总 URL 数} 个来源

## 摘要
（200 字以内的核心发现）
