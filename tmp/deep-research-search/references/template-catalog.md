# 节点模板：融合结构（产出层 + 结构层）

每个图谱节点 = **产出层**（核心判断）+ **结构层**（客观事实 F / 呈现观点 O / 关联分析 R）+ 信息缺口。

- **认知层次分类**（非数据形态）：事实=发生了什么（可验证）；观点=谁说了什么判断（带归属）；关联=为什么/怎么演变（实体间的关系）。
- **事实/观点是节点，关联是边**：关联用 `[[双链]]` 连接事实/观点，图谱才真是"图"。
- subagent 逐页填**结构层**（F/O/R + 边 + 信度），节点级填**产出层**（核心判断）。

---

## 节点结构

```markdown
# {主题节点}

## 核心判断                      ← 产出层（节点级综合，逐页时留空）
（所有页读完后，基于事实+观点+关联推导的 2-3 句核心认知。不是事实罗列，是判断。）

## 客观事实                      ← 结构层·节点（发生了什么，可验证）
### F1: {事实名称}
- 时间 / 事件 / 人物 / 地点 / 环境（5W，按相关性填，不必全有）
- 来源: [[data文件]] [信度A-E]

### F2: ...

## 呈现观点                      ← 结构层·节点（谁说了什么判断）
### O1: {谁的什么观点}
- 谁: {归属：专家/机构/媒体/匿名} | 立场·动机: {} | [信度] | 时间: {}
- 触发(因为什么): {}
- 观点: {}
- 理由(为什么这么说): {}
- 支撑: [[F1]] [[F2]]              ← 边：观点基于哪些事实
- 反例/边界: 什么情况下不成立

## 关联分析                      ← 结构层·边（为什么/怎么演变）
### R1: {名称}
- 类型: {根因|因果|张力|沉默|背景|局势|导火索|发展|结束|印证}
- 描述: {}
- 连接: [[O1]] → [[F3]] / [[他节点]]   ← 边连到哪些事实/观点/节点
- 证据 / 反例: {}

## 信息缺口
- G1: {影响判断的缺失，注明为何重要}
```

## 信度标准（绑在 F# 来源 和 O# 归属上）

| 信度 | 含义 | 判断 |
|------|------|------|
| A | 独立统计 / 3+独立源交叉确认 | 第三方统计、多源一致事实 |
| B | 权威机构 / 外媒深度 | FIFA/官方、BBC/Athletic/Reuters |
| C | 行业媒体 | 36氪/虎嗅/新浪/搜狐等有编辑审核 |
| D | 官方/企业自述 | 财报/官方公告（可能选择性呈现） |
| E | 营销/软文/自媒体 | 公关稿、水文（夸大，谨慎） |

- 事实多源交叉 → 升 A；观点出自营销号 → 降 E。
- 看原始来源，不看转载渠道（QuestMobile 经自媒体转载仍按原机构信度）。

## 关联类型清单（R# 的"类型"，按需选，不强填全部）

`根因 | 因果 | 背景 | 局势 | 导火索 | 发展原因 | 沉默原因 | 结束原因 | 张力(对立观点) | 印证(多方共识)`

- **按需填**：哪些关联在这个节点真实存在，就建哪些边。不强求全填（凑数违反"洞察宁缺毋滥"）。
- **「沉默原因」「张力」尤其重要**——前者汇聚到 smelting 的"共享沉默"，后者汇聚到"核心张力"。
- 边界模糊的信息（如"某人提出的因果判断"）→ **主类型 + 跨类要素**：主类型是关联 R，但记录"谁提出"（观点归属）。
