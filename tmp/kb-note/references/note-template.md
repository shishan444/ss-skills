# 知识笔记模板

## Frontmatter

```yaml
---
# ▼ 人维护（必填）
anchor: "（一句话核心判断）"
tags: [标签1, 标签2, 标签3]
leads: [线索词1, 线索词2, 线索词3]

# ▼ LLM 维护
aliases: [别名1, English Name]
support:
  - "[[笔记A]] 支撑理由"
resonate:
  - "[[笔记B]] 共振理由——底层共同结构"
tension:
  - "[[笔记C]] 张力所在——具体矛盾点"
instance:
  - "[[笔记D]] 实例说明——如何体现本条概念"
---
```

### 字段说明

#### 人维护字段

| 字段 | 类型 | 必填 | 职责 |
|------|------|------|------|
| `anchor` | 单行字符串 | 是 | 语义匹配——核心判断的一句话压缩 |
| `tags` | 字符串列表 | 是 | 领域粗筛——这条笔记讲的是什么（2-5 个） |
| `leads` | 字符串列表 | 是 | 联想检索——什么词会把你带到这条笔记 |

#### LLM 维护字段

| 字段 | 类型 | 必填 | 职责 |
|------|------|------|------|
| `aliases` | 字符串列表 | 否 | Obsidian 原生别名——英文名、同义词 |
| `support` | 字符串列表 | 否 | 图遍历——这条笔记为哪些笔记提供理解基础 |
| `resonate` | 字符串列表 | 否 | 图遍历——哪些笔记与这条有结构相似性 |
| `tension` | 字符串列表 | 否 | 图遍历——哪些笔记与这条存在矛盾 |
| `instance` | 字符串列表 | 否 | 图遍历——本条概念的具体表现/案例 |

### 格式约束

- 所有字段使用平铺结构，不使用嵌套字典（Obsidian 不支持）
- wikilinks 必须用引号包裹：`"[[笔记名]]"`
- 空关联类型不写该字段（不要用 `[]`，Obsidian 会将空列表转为 `null`）
- tags 控制在 2-5 个
- anchor 必须是单行字符串，不支持多个描述
- tags 和 leads 不应有重叠项

### 四种关联类型判断标准

| 类型 | 回答的问题 | 判断标准 |
|------|----------|---------|
| support | A 为 B 提供了什么基础？ | 如果 A 被推翻，B 会动摇吗？ |
| resonate | A 和 B 底层有什么共同结构？ | 跨领域能互相迁移洞察吗？ |
| tension | A 和 B 在哪里矛盾？ | 两者能同时成立吗？ |
| instance | A 在现实中长什么样？ | 能说"B 是 A 的一个例子"吗？ |

## 正文结构

每层用 `##` 标题分隔。

### `## 锚定`

核心判断的展开版。Frontmatter 里的 anchor 是压缩到极致的一句话，这里可以展开：
- 补充必要的限定条件
- 说明这个判断成立的背景
- 用自己的话重新表述，确保是消化后的理解而非照搬原文

### `## 源头`（可选）

什么情境触发了这段知识？当时在解决什么具体问题？
- 如果没有明确的触发问题（随机阅读中遇到的），跳过这一层，不要硬写
- 如果有明确触发，写下来——它帮你日后回忆"我当时为什么要学这个"
- "在一本书里读到的"是来源信息不是源头，应放在根基层

### `## 根基`

支撑锚定层判断的证据和推理。回答"凭什么这样认为"。
- 实验数据、研究来源
- 逻辑推理链条
- 个人经历或观察
- 权威观点引用

区分强证据（实验、数据）和弱证据（个人感受、未验证观点）。
每个支撑点保持简洁。这一层的功能是：日后你能判断这条知识还靠不靠谱。

### `## 疆界`

这条知识在什么条件下成立，在什么条件下失效或变形。
- 适用范围（金额大小、经验水平、文化背景等）
- 已知的例外情况
- 不确定的灰色地带

**至少 2 个边界条件。** 标注"不确定"比假装确定更有价值。
疆界不是"补充说明"，是"这条知识在哪里不好用的地图"。

### `## 脉络`

与已有知识之间的关联，带完整上下文。

格式：
```markdown
- 支撑 [[笔记名]]：完整描述支撑关系和因果链路
- 共振 [[笔记名]]：完整描述底层共同结构和区别
- 张力 [[笔记名]]：完整描述矛盾所在和思考
- 实例 [[笔记名]]：完整描述这个案例如何体现本条概念
```

Frontmatter 里的 connections 只有类型、目标和一句话理由，
脉络层保留完整的关联逻辑。LLM 会从这里提取压缩版写入 Frontmatter。

### `## 活用`

知道这条知识后，在什么场景下会做出不同的决策或判断。
- 每个场景保持一个完整的"触发条件 → 应对方式"闭环
- 如果写不出任何活用场景，标注"暂无明确活用场景，待积累"

---

## 沉积层

标题 `## 沉积`，作为正文的最后一个章节。

### 迁移记录原则

不是所有修改都需要记录。只记录理解发生了结构性变化的时刻：
- 判断变了
- 判断没变，但支撑它的理由变了
- 发现了新的疆界
- 意识到之前的理解范围过宽或过窄

措辞调整、格式修改、补充案例等维护性操作不需要记录。
新笔记的沉积层留空，等待第一次认知迁移时再添加记录。

### 迁移事件格式

```markdown
### 迁移 1 · 2026-04-30

**从**：（之前的理解——可以是一句话）

**到**：（新的理解）

**触发**：（是什么把你从"从"推到了"到"——一次阅读、一次实践、一次讨论）

**关联变化**：（是否导致了笔记之间关系的变化）
```

```markdown
tags: [认知偏差, 行为经济学, 决策]

  - "[[为什么好的谈判者不怕先让步]] 先让步创造对方的收益感，触发对方的损失厌恶"
resonate:
  - "[[沉没成本谬误]] 同源于对已投入不可回收资源的过度加权"
  - "[[禀赋效应]] 不愿放弃已拥有物，本质是损失厌恶的一种表现"

  - "[[损失厌恶可被训练消除]] 如果可训练则非硬件，与进化适应论冲突"

# 损失厌恶

## 锚定

人对损失的痛苦感受约为等量收益快乐感受的2-2.5倍。这不是一个比喻，而是一个被大量实验重复验证的稳定比例。这个不对称意味着：人在面对风险时，天然倾向于避免损失而非追求收益——哪怕两者的期望值相同。

## 源头

最初在研究"为什么大多数人投资收益低于市场平均"这个问题时，接触到行为经济学对传统理性经济人假设的挑战。损失厌恶是这个挑战中最核心的概念之一。

## 根基

- 卡尼曼和特沃斯基1979年提出前景理论，损失厌恶是核心组件之一
- 原始实验：受试者在"确定得900元"和"90%概率得1000元"之间多数选前者（风险规避），但在"确定失900元"和"90%概率失1000元"之间多数选后者（风险寻求）
- 跨文化实验重复验证率高，在不同经济水平的国家中都观察到类似比例
- 神经科学层面：损失和收益激活不同脑区，损失相关区域的激活强度约为收益区域的2倍（Tom et al., 2007）

## 疆界

- **金额边界**：小额决策中效应显著，极端大额损失时效应可能钝化
- **经验边界**：专业交易员经过长期训练后，损失厌恶程度明显低于普通人
- **群体决策边界**：群体讨论是否放大或抑制损失厌恶，目前研究结论不一致
- **文化边界**：跨文化都能观察到，但具体比例（2x还是2.5x）有文化差异

## 脉络

- 共振 [[沉没成本谬误]]：底层机制高度相似——都是对"已经投入且不可回收的资源"赋予了过高的心理权重。区别在于：沉没成本面向"过去的投入"，损失厌恶面向"未来的可能损失"
- 共振 [[禀赋效应]]：人对自己已拥有的东西估价更高，本质上也是损失厌恶——放弃拥有物 = 损失
- 支撑 [[投资决策中的非理性行为]]：解释了为什么投资者宁愿继续持有亏损股票也不愿止损
- 支撑 [[为什么好的谈判者不怕先让步]]：先让步让对方获得"收益感"，对方为了避免"失去"这个收益，更倾向于做出回报性让步
- 张力 [[理性经济人假设]]：直接挑战了理性效用最大化的前提——如果人对得失的评估是不对称的，效用计算本身就不是对称的
- 张力 [[损失厌恶可被训练消除]]：部分研究显示专业训练可以显著降低损失厌恶程度。如果它能被训练掉，那它到底是"认知硬件"还是"认知软件"？
- 实例 [[股票止损犹豫现象]]：投资者持有亏损股票不愿卖出，宁愿承担继续下跌的风险也不愿"锁定损失"——正是损失厌恶在投资场景中的典型表现

## 活用

- **投资决策**：定期问自己"如果我手持现金，我会以当前价格买入这只股票吗？"如果不会，持有它的唯一理由就是不愿锁定损失
- **产品设计**：强调"不行动会失去什么"比"行动会得到什么"更有说服力
- **自我觉察**：当发现自己在某个决策上犹豫不决时，检查是否因为"放弃 = 损失"的心理在驱动

## 沉积

### 迁移 1 · 2026-03-15

**从**：损失厌恶是一种纯认知偏差，完全非理性，应该被克服

**到**：损失厌恶可能是进化适应的产物——在生存环境中，对损失更敏感的个体存活率更高

**触发**：读到《不确定世界的理性》中关于进化心理学对认知偏差的重新解读

**关联变化**：新增 leads `进化心理学`；对 [[损失厌恶可被训练消除]] 的张力关联更加确信

### 迁移 2 · 2026-04-20

**从**：损失厌恶的2-2.5倍比例是一个跨文化的普遍常数

**到**：这个比例有文化差异和情境差异，方向可靠但具体倍数不是精确的通用参数

**触发**：在跨国产品定价策略讨论中发现亚洲市场的"限时优惠"效果不如欧美市场显著

**关联变化**：疆界层增加"文化边界"；anchor 表述从"约2倍"调整为"约2-2.5倍"
```
