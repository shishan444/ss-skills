# 正确工作范式：五个判断闸门与引导问题

本文件提供五个判断闸门、24 个条件化展开项的具体问法和澄清标准。不要逐字逐项提问；先定位核心问题的缺口，再选择对应展开项中的少量问题。

## 目录

1. 局面成立
2. 结果可验
3. 路径可信
4. 风险可控
5. 行动闭环
6. 可选复盘：可复用经验
7. 总体完整性检查

## 通用使用规则

每个展开项包含三部分：

- **何时展开**：说明它要诊断哪类缺口。
- **引导问题**：提供现实语言的入口和必要追问。每次只选 1—3 个，不要全部提问。
- **澄清标准**：说明何时已经足够，应当停止追问。

引导问题不是问卷。优先选择最能改变后续判断的问题；用户已经主动回答的内容直接整理并校验。

## 一、局面成立

**核心问题：真正需要改变的现实是什么，为什么值得现在处理？**

这一关判断工作的出发点是否真实成立，而不是只确认“有人布置了任务”。

### 1. 表面任务

**何时展开：** 需要区分原始要求和真实问题，或者对方给出的可能只是一个预设方案。

**引导问题：**

- 最初是谁要求你做什么？
- 对方原本要求交出什么，或者采取什么行动？
- 这是已经确定的目标，还是对方提出的一种解决办法？
- 如果不采用对方提出的做法，真正要求实现的结果仍然是什么？

**澄清标准：** 能准确复述原始要求，并且没有把它直接等同于真实问题。

### 2. 真实问题

**何时展开：** 只有“完成某项工作”的描述，不知道现实中究竟需要改变什么。

**引导问题：**

- 现在具体出现了什么不理想的现象？
- 这种现象发生在谁、什么场景或哪个环节？
- 有什么事实说明它确实存在？
- 目前说的是已经观察到的现象，还是对原因的推测？
- 如果任务完成了，但这个现象没有改变，是否仍然算成功？

**澄清标准：** 能描述现实中需要改变的状态，并区分已知现象和尚未证实的原因。

### 3. 对象

**何时展开：** 不知道成果为谁服务，或者不同角色可能有不同利益。

**引导问题：**

- 谁会直接使用这项工作的成果？
- 使用成果之后，希望他采取什么行动或作出什么决定？
- 最终谁会因此受益？
- 谁虽然不直接使用成果，却会承担它带来的影响、成本或变化？
- 这些角色的诉求是否冲突？

**澄清标准：** 使用者、获益者和受影响者得到区分，关键利益冲突已经暴露。

### 4. 目的

**何时展开：** 不知道为什么做、为什么现在做，或者目的仍是“完成任务”。

**引导问题：**

- 这项工作最终要支持什么行动、判断或决定？
- 为什么需要现在处理，而不是以后再做？
- 如果不做，谁的什么事情会受到影响？
- 这种后果已经发生，还是预计可能发生？

**澄清标准：** 目的指向现实中的行动、决定或变化，并能解释当前时点。

### 5. 价值

**何时展开：** 知道要做什么，却不知道做完有什么意义。

**引导问题：**

- 事情完成后，谁的什么处境会比现在更好？
- 具体会增加什么、减少什么或者避免什么？
- 这种改善与本次工作之间有什么直接关系？
- 如果没有发生这种改善，这项工作还值得做吗？

**澄清标准：** 能说明为谁创造什么变化，以及这种变化为什么有价值。

### 闸门通过表达

> 当前存在____问题。本次工作通过____，支持____采取____行动，从而改变____，其价值是____。

## 二、结果可验

**核心问题：做成什么样，看到什么才算真的发生？**

这一关区分“工作已经交付”和“交付真正产生效果”。

### 6. 交付物

**何时展开：** 不知道最终必须产生什么，或者成果形式和粒度不明确。

**引导问题：**

- 工作结束时，现实中必须出现什么可以识别的东西、决定或状态变化？
- 谁会收到或接触这个结果？
- 它需要采用什么形式，具体到什么程度？
- 如果只能保留一个最终成果，必须保留什么？

**澄清标准：** 最终结果可以被明确识别，并足以支持预期使用。

### 7. 成功标准

**何时展开：** 不同人可能对“完成”有不同理解，或标准仍是主观评价。

**引导问题：**

- 谁会判断这项工作已经达到要求？
- 他会查看哪些事实、内容或数据？
- 哪些条件必须全部满足？
- 哪些表述仍然只是“做好、完整、高质量”等个人感觉？
- 如果出现争议，依据什么作最终判断？

**澄清标准：** 完成条件具体、可观察、可检查，不依赖个人感觉。

### 8. 价值证据

**何时展开：** 交付完成不等于价值发生，或者不知道怎样回看效果。

**引导问题：**

- 交付以后，使用者需要发生什么行为，才说明成果真的被使用？
- 使用之后，什么结果或数据应该发生变化？
- 需要观察多长时间才能判断效果？
- 如何区分本次工作带来的变化和其他因素造成的变化？

**澄清标准：** 能分别说明“已经交付”的证据和“已经产生价值”的证据。

### 9. 范围

**何时展开：** 任务容易不断扩大，或者不知道哪些内容是本次目标成立所必需的。

**引导问题：**

- 为了达到本次目的，哪些内容缺少了就无法成立？
- 哪些内容虽然有用，但即使这次不做，也不妨碍核心目标？
- 与本工作相邻的问题有哪些？这次分别处理到什么位置？
- 被排除的事项由谁处理，还是暂时不处理？

**澄清标准：** 必须完成的内容、明确排除的内容和相邻事项的边界清楚。

### 10. 取舍

**何时展开：** 时间、质量、内容和成本无法同时满足。没有现实冲突时跳过。

**引导问题：**

- 当前哪些要求可能无法同时满足？
- 如果必须牺牲一项，首先保住什么？
- 哪些内容可以降低标准、缩小范围或推迟完成？
- 哪些底线无论如何不能突破？
- 谁有权决定最终取舍？

**澄清标准：** 冲突发生时，优先顺序、可牺牲项、不可突破的底线和决策权明确。

### 闸门通过表达

> 我们将向____交付____，支持其____。满足____条件算完成；之后看到____，才说明价值发生。本次必须包含____，明确不包含____；发生冲突时优先保住____。

## 三、路径可信

**核心问题：凭什么认为这些行动能够带来目标结果？**

这一关验证行动和结果之间的因果关系，不把待办事项数量当作路径质量。

### 11. 工作路径

**何时展开：** 只有行动清单，无法解释为什么这些行动会带来结果。

**引导问题：**

- 当前状态与目标状态之间，最关键的差距是什么？
- 要消除这些差距，现实中必须依次发生哪些变化？
- 每项行动准备促成哪一种变化？
- 如果某个步骤完成了，却没有推动下一步，它还有必要吗？
- 哪个环节最可能存在因果跳跃？

**澄清标准：** 能说明“行动—中间变化—最终结果”的连接，而不是只列待办事项。

### 12. 里程碑

**何时展开：** 工作周期较长，不知道何时可以进入下一阶段。

**引导问题：**

- 出现什么可验证的结果，才说明当前阶段结束？
- 谁来确认这个结果？
- 最晚什么时候需要确认？
- 未达到什么条件时，不应进入下一阶段？

**澄清标准：** 每个必要里程碑都是可验证的事件或阶段结果，并有时间和确认人。

### 13. 依赖

**何时展开：** 推进需要外部信息、资源、权限或决定。

**引导问题：**

- 哪些事情不是你单独能够完成或决定的？
- 需要谁提供什么信息、资源、权限或决定？
- 最晚什么时候必须获得？
- 如果没有按时获得，哪部分工作会被阻断？
- 有没有临时替代方案？

**澄清标准：** 关键依赖的内容、提供者、时限、缺失影响和替代办法明确。

### 14. 使用场景

**何时展开：** 方案看似完整，却不知道成果怎样进入真实工作。

**引导问题：**

- 谁会在什么时间和场合使用成果？
- 他拿到成果后，具体会做什么？
- 使用前还需要满足什么条件？
- 当前设计是否符合他的实际工作方式？
- 如果他不使用，最可能是什么原因？

**澄清标准：** 成果能够进入具体使用场景，并促成预期行动。

### 闸门通过表达

> 从当前状态到目标结果，需要依次发生____。我们采取____行动促成这些变化，以____作为阶段完成标志。路径依赖____，最终由____在____场景中使用成果。

## 四、风险可控

**核心问题：哪个关键判断最可能错，如何尽早发现？**

这一关优先寻找可能使方案失效或造成重大返工的不确定性，不追求列出所有坏事。

### 15. 假设

**何时展开：** 方案依赖尚未证实的判断，或事实与推断混在一起。

**引导问题：**

- 当前方案默认相信什么是真的？
- 其中哪些只是判断，还没有事实证明？
- 哪项判断如果错误，会让方案失去价值或造成最大返工？
- 这个假设影响哪一项行动或决定？
- 它错误后是否容易补救？

**澄清标准：** 事实与假设得到区分，并找到最影响方案成立的关键假设。

### 16. 最小验证

**何时展开：** 关键假设不宜等到全面执行后才验证。

**引导问题：**

- 最少做什么，就能获得判断这个假设所需的证据？
- 能否先找少量用户、样本或场景进行测试？
- 需要看到什么结果，才能继续当前方案？
- 出现什么结果时应修改或停止？
- 验证成本是否明显低于直接全面执行？

**澄清标准：** 验证动作、所需证据以及继续、修改、停止的条件明确。

### 17. 风险

**何时展开：** 存在可能阻止目标实现的事件，需要提前识别信号和应对办法。

**引导问题：**

- 在目前路径中，什么事情可能使目标无法实现？
- 它与“尚未证实的判断”有什么区别？
- 风险发生前可能出现什么信号？
- 一旦发生，会影响时间、质量、范围还是价值？
- 可以怎样预防？发生后怎样降低损失？

**澄清标准：** 主要风险具有预警信号、影响判断和具体应对办法。

### 18. 升级条件

**何时展开：** 执行者可能遇到无权决定或影响重大的变化。

**引导问题：**

- 出现什么情况时，继续自行推进会造成更大风险？
- 哪些变化必须立即让决策者知道？
- 延误多久、成本增加多少或结论发生什么变化时必须升级？
- 应该通知谁，最晚什么时候通知？
- 升级时需要对方决定什么？

**澄清标准：** 升级触发条件、通知对象、时限和所需决策明确。

### 闸门通过表达

> 当前最关键的未证实判断是____。如果错误，将导致____。我们先通过____验证；看到____则继续，看到____则调整或停止。主要风险是____，出现____时必须向____升级。

## 五、行动闭环

**核心问题：现在谁决定、谁行动，何时根据什么调整？**

这一关把前面的判断转化为当前行动和持续反馈。

### 19. 角色

**何时展开：** 责任、决策权、输入或验收关系不明确。

**引导问题：**

- 谁对最终结果负责？
- 谁负责组织和完成当前行动？
- 遇到重大分歧时谁作最终决定？
- 谁提供必要输入？
- 谁依据标准确认完成？
- 是否存在多人都能决定，或者无人真正负责的情况？

**澄清标准：** 结果责任、执行、决策、输入和验收角色清楚，没有责任空缺或决策冲突。

### 20. 状态

**何时展开：** 只能说出完成百分比，却不知道真实进展和阻塞点。

**引导问题：**

- 目前已经确认了哪些事实或结果？
- 还有哪些关键问题没有答案？
- 当前具体卡在哪里？
- 是什么阻止了下一步？
- 现在最需要完成的一个动作是什么？

**澄清标准：** 已确认、未确定、阻塞点和下一步能够被分别描述。

### 21. 待决策事项

**何时展开：** 工作卡在选择、授权或重大取舍上。

**引导问题：**

- 现在具体需要决定什么？
- 有哪些现实可选方案？
- 谁有权作出决定？
- 最晚什么时候必须决定？
- 你的建议是什么，依据是什么？
- 延迟决定会影响什么？

**澄清标准：** 决策内容、决策者、时限、建议和延误后果明确。

### 22. 检查点

**何时展开：** 需要阶段性对齐，但不知道为什么开会或检查什么。

**引导问题：**

- 下一次对齐要验证什么事实，或者作出什么决定？
- 在此之前谁需要准备什么？
- 哪些人必须参加？
- 哪些问题本次明确不讨论？
- 检查后可能产生哪些下一步？

**澄清标准：** 检查点具有明确时间、参与者、输入和判断目的。

### 23. 验收

**何时展开：** 不知道谁来结束工作，或者修改可能无限持续。

**引导问题：**

- 谁最终确认交付完成？
- 他依据哪些已经约定的成功标准验收？
- 不符合时由谁修改，修改到什么程度？
- 哪些意见必须处理，哪些属于新增需求？
- 达到什么条件后不再继续修改？

**澄清标准：** 验收人、验收依据、修改责任和结束条件明确。

### 24. 结果回看

**何时展开：** 交付结束后还需要判断是否被使用、是否产生结果和价值。

**引导问题：**

- 什么时候检查成果是否被实际使用？
- 什么时候检查它是否产生了预期结果？
- 什么时候检查最终价值是否发生？
- 每次回看查看什么行为、事实或数据？
- 谁负责回看？结果不符合预期时怎么办？

**澄清标准：** 使用、结果和价值三个层次都有必要的回看时间、证据和负责人。

### 闸门通过表达

> 当前由____决定____，由____执行____。已经确认____，仍需解决____，下一步是____。我们将在____检查____；由____按照____验收，并在____回看实际结果，根据证据决定继续、调整或结束。

## 六、可选复盘：可复用经验

**何时展开：** 工作结束、关键判断被修正，或者出现明显成功、失败和返工时。它不是五个闸门通过的必要条件。

**引导问题：**

- 这次哪一种做法在相似任务中可以直接复用？
- 哪个判断或动作造成了不必要的返工？
- 下次遇到什么信号时，应提前采取什么行动？
- 能否把经验写成“当……时，就……”的行动规则？

**澄清标准：** 经验被写成可以在相似情境中触发的行动规则，而不是“加强沟通、提高效率”等感想。

## 七、总体完整性检查

最终应形成一条完整主链：

> 当前现实 → 工作目的 → 目标结果 → 行动路径 → 关键假设 → 当前行动 → 证据反馈 → 结果与价值

并能回答：

- 为什么值得现在处理？
- 要改变谁的什么现实？
- 看到什么算完成，看到什么才算产生价值？
- 行动为什么可能有效？
- 哪个关键判断最可能错，如何尽早发现？
- 谁决定、谁执行、谁验收？
- 当前在哪里，下一步是什么？
- 根据什么证据继续、调整、停止或升级？

无法连接的部分是逻辑断点。返回对应闸门，只展开能够解释断点的要素。
