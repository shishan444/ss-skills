# 探测引导话术与追问策略

> `/st-profile` 三轮探测中的对话引导细节。

## 第一轮：领域扫描引导

### 标准开场
> 我需要了解你认真投入过精力的领域，不限专业还是业余，不限技术还是人文。对每个领域，只需要告诉我：
> 1. 你做了什么（读过书 / 做过项目 / 写过东西 / 教过人 / 长期实践）
> 2. 大概投入了多长时间
>
> 不需要评估你理解到什么程度，只告诉我你的行为轨迹就行。

### 如果用户列举太少（<3 个领域）
> 除了这些，有没有一些你花过时间但没觉得值得一提的？比如某个兴趣、某个你帮别人解决过问题的领域、某个你经常关注的话题？

### 如果用户只给领域名没给行为
> {领域名}这个领域，你具体做了什么？是读过相关的书、做过项目、还是以其他方式投入的？

## 第二轮：教学探测引导

### 核心概念选择原则
- 选择该领域**最核心、最有解释力**的概念，而非最简单或最复杂的
- 避免选择纯术语定义（如"什么是 API"），选择有机制的概念（如"REST 的无状态约束意味着什么"）
- 如果不确定领域核心概念，可以问："这个领域里，你觉得最能代表你理解的东西是什么？"

### 教学请求模板
> 假设我是个聪明的门外汉，对这个领域一无所知但学习能力强。请给我讲清楚 **[概念X]**。不用术语堆砌，用你觉得最能让一个聪明人理解的方式讲。

### 教学过程中的引导
- **不插话**：让学习者完整讲完，不要中途纠正或补充
- **记录原始表述**：学习者说的原话是锚点的核心材料，不能改写
- 如果学习者卡住或明显在回忆而非理解：
  > 没关系，用自己的理解讲就好，不一定要完整准确。

### 教学结束后的确认
> 谢谢。你刚才的讲解我记下了。接下来聊下一个领域。

不要当场评价教学质量。评价是 AI 内部做的事，不暴露给学习者。

## 第三轮：反向追问策略

### 选择追问对象
从第二轮中挑选**学习者展现了信号 C（自然连接）**的片段优先追问。这些是最有价值的隐性结构。

### 追问类型

**类比来源追问**（当学习者用了类比）：
> 你刚才讲 [概念] 的时候用了 [类比]。你是怎么想到用这个来解释的？在你脑子里，它们哪里是一样的？

**边界追问**（当学习者没指出边界时）：
> 你刚才讲得很清楚。我好奇的是——[概念]在什么情况下会失效？或者说，什么时候你刚才的解释不适用了？

**跨域映射追问**（当学习者自然连接了其他领域时）：
> 你刚才不自觉地把它和 [领域Y] 联系起来了。你觉得这个联系的核心是什么——它们共享了什么底层结构？

**反面追问**（当学习者讲得特别流畅时）：
> 你讲得很顺畅。如果让你指出这个概念最容易被误解的地方，你觉得是什么？

**变体追问**（当学习者的理解看起来比较独到时）：
> 你刚才的解释和教科书上的不太一样。你觉得你的理解和标准表述之间，哪里不同？为什么你选择这样理解？

### 追问原则
1. **不诱导**：不要在问题中暗示答案（"你是不是因为 X 所以这么想？"）
2. **追问结构而非内容**：问"为什么想到用这个类比"，不问"这个类比对不对"
3. **保持自然**：追问是讨论，不是审讯。语气温和好奇
4. **适可而止**：每个领域最多追问 2-3 个问题，不要把学习者问累

## 观察记录格式

教学探测中，对每个领域的观察记录格式：

```
### {领域}: {核心概念}
信号A(剥离术语): {通过/未通过} — {具体观察}
信号B(指出边界): {通过/未通过} — {具体观察}
信号C(自然连接): {通过/未通过} — {具体观察}
原始表述摘要: "{学习者的关键原话}"
提取的锚点: {概念名} — {可挂载的知识类型}
认知桥梁: {如有}
```

## 画像合成的锚点提取规则

### 从教学探测中提取锚点
只有**同时通过信号 A 和信号 B** 的概念节点才能进入"深度锚点"：
- 信号 A 通过 = 学习者确实理解了，不是在背术语
- 信号 B 通过 = 学习者的理解有边界意识，不是模糊的大概

只通过信号 A 但没通过信号 B 的：放入"待验证领域"而非"深度锚点"。
三个信号都通过的：优先级最高的锚点。

### 从追问中提取桥梁
只提取学习者在**追问中确认**的跨域映射。AI 猜测但学习者未确认的映射不写入"认知桥梁"。

### 思维偏好推断
从教学行为中推断，不直接问：
- 学习者解释时倾向于用类比 → 切入方式：类比
- 学习者解释时倾向于描述机制 → 切入方式：机制
- 学习者在某类问题上特别敏锐 → 记录具体问题类型
