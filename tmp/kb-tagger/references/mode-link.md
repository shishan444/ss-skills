# link 模式：增量双向链接

## 触发

用户指定一篇中心笔记，要求建立笔记间的关联。

## 前置加载

执行前必须已读取 references/frontmatter-schema.md。

---

## 流程

### Step 1：扫描全库

运行扫描脚本：

```bash
python scripts/kb_scan.py <知识库目录>
```

脚本输出全部笔记的 frontmatter 摘要文本。基于此文本进行后续分析。

---

### Step 2：读取中心笔记

使用 Read 工具读取中心笔记全文。记录已有的 connections（不重复建立）。

---

### Step 3：粗筛候选

基于扫描输出的 anchor + tags + leads，筛选与中心笔记可能相关的候选笔记。

排除：
- 已存在 connection 的笔记
- 中心笔记自身
- 没有任何语义交集的笔记

---

### Step 4：深度评估

对粗筛命中的候选笔记，使用 Read 工具读取全文。逐条评估关系类型。

**四种关系判断标准**：

| 关系 | 判断标准 | 方向 |
|------|---------|------|
| support | A 被推翻 → B 会动摇吗？ | A 支持 B |
| resonate | A 和 B 共享深层结构？洞察能跨域迁移？ | 双向 |
| tension | A 和 B 存在矛盾？两者能同时为真？ | 双向 |
| instance | 能说"B 是 A 的例子"吗？ | A 抽象，B 具体 |

**避免浅层关联。** 只是"属于同一领域"或"提到同一个词"不构成关系。必须有实质性语义关联——读完两条笔记后能说出"它们之间有 X 关系，因为 Y"。

每条链接只属于一种类型。如果同时满足多种，选最精确的那个（instance > support > resonate > tension）。

---

### Step 5：生成链接提案

格式：

```
[中心笔记] --support--> [笔记B]
  理由：中心笔记中的 X 概念为笔记 B 的 Y 论点提供了理论基础

[中心笔记] --resonate--> [笔记C]
  理由：两者都体现了"Z"这一深层模式，洞察可以互迁
```

展示提案供用户审核。用户可以接受全部或逐条确认/拒绝。

---

### Step 6：双向写入

用户确认后，对每条链接执行双向写入。

**frontmatter 写入**：
- 在中心笔记的对应关系字段（support/resonate/tension/instance）中追加 `"[[目标笔记]] 理由"`
- 在目标笔记的对应关系字段中追加 `"[[中心笔记]] 理由"`

**body 脉络段落写入**：

笔记 body 的 `## 脉络` 段格式：
```
## 脉络
- support [[笔记名]]: 完整描述
- resonate [[笔记名]]: 完整描述
```

- 在中心笔记的 `## 脉络` 段末尾追加对应行
- 在目标笔记的 `## 脉络` 段末尾追加对应行

如果笔记没有 `## 脉络` 段 → 在 `## 活用` 段之前插入新的 `## 脉络` 段。
如果笔记没有 `## 活用` 段 → 在 `## 疆界` 段之后、`## 沉积` 段之前插入。

---

### 双向写入检查清单

每条链接执行后，确认：
- [ ] 中心笔记 frontmatter connections 已更新
- [ ] 目标笔记 frontmatter connections 已更新
- [ ] 中心笔记 body 脉络已追加
- [ ] 目标笔记 body 脉络已追加
- [ ] 已有链接未被修改或删除
- [ ] 理由文本具体清晰，不是泛泛的"相关"

---

### 安全规则

- 只追加，不删除已有链接
- 写入 body 时只在 `## 脉络` 段追加，不修改其他段落
- connections 格式：`"[[笔记名]] 一句话理由"`
- 脉络格式：`- type [[笔记名]]: 完整描述`
