# tag-normalize 模式：标签规范化

## 触发

用户要求整理标签体系、合并重复标签、全库标签一致性检查。

## 流程

---

### Step 1：扫描全库标签

使用 Grep 工具扫描知识库目录下所有 .md 文件的 frontmatter，提取 tags 和 leads 字段。

收集所有唯一标签及其出现位置（哪些文件使用了该标签）。

**不可跳过此步。** 不扫描全库就操作，后续标签必然不一致。

---

### Step 2：分析问题

识别以下类型的问题：

**同义不同名**：两个标签指向同一概念
- `前端` vs `frontend` vs `前端开发`
- `机器学习` vs `ML` vs `machine-learning`
- 判定方法：两个标签能否互换使用而不改变含义？

**粒度不当**：
- 太泛（`技术`、`笔记`）→ 任何文件都能用，无法区分，考虑移除或替换
- 太细（只有一个文件使用的超长标签）→ 考虑合并到更通用的标签

**tags / leads 重叠**：
- 同一个词同时出现在某个文件的 tags 和 leads 中 → 按 frontmatter-schema.md 的规则决定归属（优先归入 leads）

---

### Step 3：生成规范化方案

将发现的问题整理为方案，格式：

```
1. 合并：`前端` + `frontend` → 统一为 `前端`（涉及 5 个文件）
2. 重命名：`machine-learning` → `机器学习`（涉及 3 个文件）
3. 移除：`技术`（太泛，无区分度，涉及 2 个文件）
4. 重叠修正：文件 X 中 `认知科学` 同时在 tags 和 leads → 移至 tags
```

方案按影响范围排序（影响大的优先展示）。

---

### Step 4：审核与执行

展示方案供用户审核。用户可以选择接受全部或部分。确认后逐文件执行。

**执行规则**：
- 使用 Edit 工具逐文件修改 frontmatter
- 只修改 tags 和 leads 字段
- 不触碰 anchor、aliases、connections
- 保留分组注释结构
