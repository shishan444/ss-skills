# st-connect — 跨域连接发现

扫描已学概念，寻找不同来源、不同领域之间的结构同构。它的目标不是“多联想”，而是找到能帮助用户迁移理解的骨架。

## 边界

`st-connect` 只提出和记录连接，不学习新概念，不提升 mastery，不自动改写笔记正文。

写入 `profile.md` 的连接必须经过用户确认。AI 自己觉得像，只能作为候选。

## 前置检查

1. 读取 `learning/learner/model.json`，筛选 `mastery >= 2` 的概念。
2. 读取相关 `learning/extractions/*.md`。
3. 读取 `learning/learner/profile.md`，获取已有锚点和已确认桥梁。
4. 如存在 `learning/notes/*.md`，读取相关笔记中的“何时用得上”和“跨域映射”。

如果可用概念少于 3 个，输出：

```text
已掌握概念还太少（{N} 个），暂时不适合做跨域连接。
继续用 /st-study 学更多概念后再运行 /st-connect。
```

## 候选连接生成

对 `mastery >= 2` 的概念做候选配对，优先：

- 不同 source-id。
- 不同领域或材料类型。
- 拓展层出现相似结构。
- 用户笔记中“何时用得上”指向相近问题。
- profile 中已有锚点能同时连接两者。

概念很多时不要全量展示。先内部排序，最多提出 5 个最强候选。

内部匹配和写回误解时以 `concept_id` 为准；概念名只用于展示。遇到同名概念，必须同时显示 source-id。

## 同构判定标准

一个连接必须同时满足：

1. 共享骨架：两个概念的底层运作机制相同或高度相似。
2. 可映射：能明确列出 A 的要素 X 对应 B 的要素 Y。
3. 有迁移价值：这个连接能帮用户在一个领域借用另一个领域的判断框架。
4. 非表面相似：不是因为都出现“系统、反馈、优化、循环”等泛词。

无效连接示例：

- “机器学习和心理学都在学习。”
- “两个概念都和系统有关。”
- “经济周期和情绪波动都上下起伏。”

有效连接的形式：

```text
共享骨架: 过去形成的约束会限制当前可选动作。
映射:
- 闭包中的词法环境 <-> 路径依赖中的历史制度条件
- 函数后续行为范围 <-> 组织后续选择空间
迁移价值: 当你看到一个系统“明明可以换路却很难换”时，可以同时调用这两个概念。
```

## 呈现候选

逐个呈现，不要一次倒出太多：

```text
━━ 候选连接 {N}/{总数} ━━

「{概念A}」({来源A}) <-> 「{概念B}」({来源B})

共享骨架:
{一句话说明}

映射:
- A 的 {要素X} <-> B 的 {要素Y}
- A 的 {要素P} <-> B 的 {要素Q}

为什么对你有用:
{迁移价值}

你觉得这个连接成立吗？如果成立，你能想到第三个也有同样骨架的概念吗？
```

## 用户确认

用户确认成立：

- 写入 `learning/learner/profile.md` 的“认知桥梁”。
- 如果用户补充第三个概念，也一并记录。
- 标注来源：`st-connect`。

用户否决：

- 不写入已确认桥梁。
- 询问一句原因。
- 如果否决原因揭示了某个概念的误解，追加到 `model.json` 对应概念的 `misconceptions`。

用户不确定：

- 写入“待确认桥梁”，不要当成已确认。

## profile 写入格式

```markdown
- {概念A} <-> {概念B}
  - 共享骨架: {结构}
  - 映射:
    - {A要素} <-> {B要素}
  - 迁移价值: {价值}
  - 用户确认: 是
  - 来源: st-connect
  - 日期: YYYY-MM-DD
```

保留已有桥梁。发现重复时合并，不要重复写。

## 连接报告

```text
━━ 连接报告 ━━
已扫描概念: {N}
候选连接: {X}

- {桥梁摘要}
```

## 自动建议

`st-study` 或 `st-review` 可以在以下情况建议用户运行 `/st-connect`：

- 完成一个新 source 的主要概念。
- 用户在学习中主动提出多个跨域类比。
- 某个概念复习表现很强，并出现迁移表达。
