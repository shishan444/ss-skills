# 小说阶段质量review

每个剧情阶段写完后的质量门禁。它的职责不是夸奖章节，也不是重新设计整本书，而是用证据判断：当前阶段能不能进入下一阶段。

核心原则：**review 必须先有证据，再有判断。** 既有 review、作者印象、上一轮结论都不能直接当证据；章节被重写后，旧结论可能已经失效。

## 职责边界

- 负责：定量扫描、精读审计、P0/P1/P2分级、修复方案、必要时精准修复、修复后验证。默认先产出 review；只有用户要求修复，或当前任务明确进入 P0/P1 修复循环时，才修改章节正文。
- 不负责：重新选题或重做完整世界观；这类问题交给 `novel-concept-design` 或 `novel-worldbuilding`。
- 不负责：做最终润色合并；P0/P1清零后交给 `novel-polish`。
- 如果问题来自章节正文，优先精准 patch；如果问题来自设定或大纲断裂，只修改必要的设定/状态文件，不顺手重构无关内容。

## 输入

- 本阶段全部章节（drafts/chapter-{NN}.md ~ chapter-{MM}.md）
- 全部设定文件（concept/worldbuilding/characters/character-ledger/outline/tone-guide/emotion-style-map/writing-guide）
- front-30-hook-map.md（若review范围涉及Ch1-30，必读）
- foreshadow-tracker.md + expectation-list.md
- 共享参考：`../写作经验库/index.md`、`../写作经验库/通用总结/writing-craft-summary.md`、`../创作方法论/foreshadow-model.md`、`../创作方法论/expectation-model.md`

## 输出

- act{N}-review.md（review报告，含P0/P1/P2分级）
- 修复后的章节文件（如需要）
- 更新后的设定文件（如需要）
- 更新后的foreshadow-tracker.md和expectation-list.md
- 更新后的character-ledger.md（如涉及角色状态/关系变化/偏离警报）

review报告必须包含：

```markdown
## 结论
- 是否可进入下一阶段：
- P0数量：
- P1数量：
- 最大风险：
- 角色漂移结论：
- 情绪路由兑现结论：
- 冷启动/追读钩子结论：
- 写作经验转化结论：

## 证据
- 定量数据：
- 精读样本：
- 与上一阶段对比：
- 旧review失效项：

## 问题清单
| 级别 | 问题 | 证据 | 影响 | 修复动作 |

## 修复记录
| 轮次 | 修复内容 | 验证结果 | 新增问题 |

## 最新状态
- P0：
- P1：
- character-ledger.md更新状态：
- 下一阶段必须注意：
```

## Step 0：定量数据采集（精读前先跑）

在精读任何章节之前，先用工具采集客观基线数据。这些数据能暴露光靠精读容易漏掉的系统性问题（如章节缩水、标志性短语泛滥、风格漂移）。

> ⚠️ **既有 review 的结论可能基于历史版本。** 小说的章节文件经常被重写，但 review 报告不会自动更新。精读前必须用 Step 0 定量数据验证既有 review 的核心判断是否仍成立——如果定量数据与既有 review 矛盾，**以定量数据为准**，并在新 review 中标注既有 review 的哪些结论已过时。

### 0.1 章节体量分析

用 terminal 对本阶段全部章节跑 `wc -l` + `wc -c`，输出每章行数和字节数：

```bash
cd stories/<项目>/drafts && for i in $(seq NN MM); do f="chapter-${i}.md"; if [ -f "$f" ]; then echo "Ch$i: $(wc -l < "$f")L $(wc -c < "$f")B"; fi; done
```

**看什么**：
- 体量急剧下降的章节群（如某几章突然只有平均值的一半）→ 可能是空壳章节，需重点审查
- 体量趋势（整体在缩水还是在膨胀）→ 缩水可能是作者偷懒/工具切换/风格退化
- 与上一阶段的体量对比 → 跨阶段对比能发现系统性变化

### 0.2 标志性短语频次分析

用 `search_files target='content' output_mode='count'` 统计角色标志性口头禅/收尾语的频次：

如果当前环境没有 `search_files`，用 `rg -n` 或脚本统计等价完成；关键是拿到每章命中次数和跨阶段密度对比，不是执着于某个工具名。

- 搜索主角口头禅（如"好家伙""离谱"等，从 tone-guide/characters.md 获取）
- 搜索公式化收尾语（如"一步一步""记住了"等，从已读章节中归纳）
- **跨阶段对比**：对本阶段和上一阶段各跑一次，看密度变化

**看什么**：
- 标志性短语从「偶尔出现的笑点」变成「每章出现N次的填充词」→ P0级退化信号
- 收尾语公式化（如连续多章以相同短语结尾）→ P1级审美疲劳

### 0.3 跨阶段风格对比

精读本阶段采样章节后，**回头读2-3章上一阶段的章节**（尤其是上一阶段review赞扬过的章节），对比：
- 句式节奏是否一致（如从正常散文变成碎片化断句）
- 内心独白的呈现方式是否变化（如从"嘴上一套心里一套的反差"变成"纯喃喃自语"）
- 信息密度是否变化（如同样篇幅里实际推进的剧情变少）

**这个对比是发现"风格漂移"P0问题的最可靠方法。** 单独看本阶段章节时，碎片化/公式化可能不易察觉；但跟上一阶段对比时差异会非常明显。

### 0.4 破折号/碎片化密度分析（最高危 P0 探测器）

用 terminal 统计每章破折号（——）数量与字节量，计算"破折号/KB"密度：

```bash
cd stories/<项目>/drafts && for f in chapter-*.md; do
  n="${f#chapter-}"; n="${n%.md}"; n=$((10#$n))
  b=$(wc -c < "$f"); d=$(grep -o "——" "$f" 2>/dev/null | wc -l | tr -d ' ')
  printf "%d\t%s\t%s\n" "$n" "$b" "$d"
done | awk -F'\t' '{dpk=$3*1024/$2; printf "Ch%d: %sB %s dashes (%.1f/KB)\n", $1, $2, $3, dpk}'
```

**看什么**：
- 正常中文散文基线：≤10 破折号/KB
- 碎片化断奏体信号：>20 破折号/KB（每个词之间加破折号，句子被拆成碎片）
- **这是比短语频次更底层的退化探测器**——它直接测量句子结构的变化，而非填充词频率
- **跨区间对比**：如果某个区间密度突然飙升（如从 8→35），就是 P0 级文体突变
- **这是唯一能推翻既有 review 核心判断的客观指标**：精读受 confirmation bias 影响，但破折号密度不会骗人

**判定标准**：≤10/KB = 正常散文；10-20/KB = 偏高（抒情/蒙太奇可接受）；>20/KB = 碎片化断奏体（P0）；>40/KB = 极端碎片化。

**实战教训**：破折号密度分析曾推翻一份既有 review 的核心 P0 判断——review 声称 Ch107-160 是碎片化断奏体（引用了 Ch107 开篇文本为证），但定量分析显示该区间密度仅 1.84/KB（正常散文）。review 引用的文本来自已被重写的旧版本。真正的碎片化在 Ch61-106（35.2/KB），是一个此前从未被 review 覆盖的盲区。

### 0.5 情绪路由覆盖分析

读取 outline.md / emotion-style-map.md / writing-guide.md，抽取本阶段每章的剧情功能、读者期待、目标体感、呈现模式、爽点阶段和章长策略。若旧项目缺少这些字段，先用章节实际内容反推一个临时表，并在问题清单中标记“情绪路由缺失”。

**看什么**：
- 关键爽点、关系定调、死亡告别、重大反转是否被标为完整展开
- 是否连续多章使用同一种呈现模式，导致读者疲劳
- 是否出现“目标体感”和正文执行相反：该爽的写平、该压的写散、该烟火气的只有流水账、该过渡的硬写成长章
- 章长是否服务剧情密度，而不是机械平均

### 0.6 冷启动与前30章钩子覆盖分析

如果review范围涉及Ch1-30，读取 concept.md / outline.md / front-30-hook-map.md，抽取每章的设定入场方式、即时收益/冲突、章末短钩子、中钩子归属和长钩子推进。若缺 front-30-hook-map.md，标记P1；若前三章无法证明“设定入场→限制加压→小爽点兑换”，标记P0。

**看什么**：
- 前三章是否让核心设定在行动中产生效果，而不是传统文学式慢慢铺氛围
- Ch1是否有具体追读画面和即时反应，Ch2是否让限制/代价卡住主角，Ch3是否交付第一个小爽点
- 前30章是否每章都有短钩子，每3-5章有中钩子，持续推进至少1条长钩子
- 读者是否能在前30章形成“这本书后面大概会怎么爽”的整体印象

## 八维度检查

### Round A：机制一致性

检查核心机制（外挂/能力/设定）在本阶段的使用中是否自洽：

- [ ] 能量/充能规则一致（充能速度/消耗量/是否跟设定一致）
- [ ] 空间/能力成长节奏跟concept.md的成长表一致
- [ ] 穿越触发条件一致
- [ ] 时间线自洽（两个世界的时间同步/历史事件按真实时间线发生/人物年龄对得上）
- [ ] 限制规则没有被违反（如果有"不能做X"的限制，本阶段有没有违反）

**发现P0的条件**：机制矛盾会导致后续剧情崩塌的问题
**发现P1的条件**：机制有小矛盾但不影响主线

### Round B：人物行为合理性

检查所有角色在本阶段的行为是否合理：

- [ ] 主角的行为跟他的性格/利益一致（有没有突然变聪明/突然变蠢）
- [ ] 配角的行为跟他的利益线一致（有没有为了剧情需要让配角做不合理的事）
- [ ] 反派/对手的智商在线（有没有让对手犯蠢来让主角赢）
- [ ] 女性角色有主体性（不是被动的花瓶/工具人）
- [ ] 新出场的角色有合理的出场理由

**发现P0的条件**：角色行为严重不合理（如对手莫名其妙犯蠢让主角赢）
**发现P1的条件**：角色行为有小瑕疵但不致命

### Round C：写作手法质量

这是最关键的维度。检查写作质量是否达标。分两个层级检查：

#### C-0：情绪路由兑现检查（逐章对照）

对每章建立“承诺 vs 正文”的对照：

```
| 章节 | 剧情功能 | 读者期待 | 目标体感 | 呈现模式 | 爽点阶段 | 章长策略 | 正文证据 | 结论 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
```

检查重点：
- **爽点兑换**：是否有铺垫回收、释放瞬间、见证者反应、结果反馈和余波？如果只有“主角赢了/钱到账了/对方服了”，不合格。
- **压迫/危机**：是否让选择变窄、信息收紧、代价具体？如果只写“气氛压抑/情况危险”，不合格。
- **烟火气/温情**：是否用物件、饭菜、声音、称呼、手上动作承载关系变化？如果只有抽象温暖或流水账，不合格。
- **轻松/搞笑**：笑点是否来自认知错位、反差或误判？如果靠角色犯蠢或口头禅刷屏，不合格。
- **过渡/压缩**：低密度节点是否压缩并交代状态变化？如果为了字数硬扩写，不合格。
- **稳定声音**：不同呈现模式切换时，叙述距离、人物语言底色和标点边界是否仍像同一本书？

**发现P0的条件**：核心爽点/死亡告别/关系定调/重大反转的目标体感没有兑现，导致读者体验失败；或整阶段情绪模式严重单一，与 tone-guide/emotion-style-map 脱节。
**发现P1的条件**：若干普通章节呈现模式选择不当，影响阅读但不破坏主线；或 emotion-style-map 缺字段导致后续写作不可控。
**发现P2的条件**：个别段落颗粒度不匹配，可在润色中调整。

#### C-1：单章质量检查（逐章标准）

- [ ] **决策过程完整展示**：有没有"他决定了"这种跳过决策的写法？关键选择是否展示了完整决策链（触发→本能抗拒→拉锯→身体先动→事后反噬）？
- [ ] **氛围有感官细节**：有没有用抽象形容词（压抑/恐怖/温暖）代替具体细节？关键场景是否有视觉/听觉/触觉/嗅觉的多感官描写？
- [ ] **内心独白占比30-50%**：主角的内心吐槽/感知/拉锯是否足够？还是只写"发生了什么"不写"主角怎么想的"？
- [ ] **像小说还是像作文**：有没有大段说明/解释/背景介绍？信息是通过场景展示的还是通过旁白讲述的？
- [ ] **对话有层次**：角色说的话有没有表面意思vs真实意图的区别？还是直来直去？
- [ ] **搞笑来自认知错位**：笑点是来自两个世界的认知差还是来自角色犯蠢？
- [ ] **写作经验已转化**：本阶段主要场景是否能看到写作经验库的执行结果？例如危机有选择收窄和代价，爽点有过程，群像有多目标冲突，谈判有底牌和假话。
- [ ] **情绪路由已兑现**：本章目标体感是否能从正文证据中读出来，而不是只在大纲里写了标签？

#### C-2：系统性退化检查（跨章/跨阶段标准）—— Step 0 数据驱动

这是 C-1 无法覆盖的维度：单章可能都"还行"，但整个阶段的写作方式已经偏离了设定和上一阶段的基线。

- [ ] **文体一致性**：本阶段的句式节奏/叙述密度是否与上一阶段一致？有没有出现全新的文体（如从正常散文突变为"每个词之间加破折号"的碎片化断奏体）？**这是最高危的P0——一旦整阶段文体突变，54章全部受影响。**
- [ ] **标志性短语密度**（Step 0.2数据）：主角口头禅是否从"偶尔出现的笑点收尾"退化成"机械填充词"？如果频次是上一阶段的3倍以上→P0
- [ ] **章节体量异常**（Step 0.1数据）：是否有连续多章体量急剧缩水（<平均值的60%）？→ 可能是空壳章节，需检查是否应该合并或扩写
- [ ] **公式化结构**：是否有大量章节使用相同的结构模板（如固定收尾模式/固定的"事件→好家伙→笔记本反思→合上本子→一步一步"流程）？
- [ ] **格调漂移**：搞笑比重是否与tone-guide设定一致？如果设定是"搞笑+奋斗"但实际搞笑元素接近0%→P0（与设定严重脱节）
- [ ] **动态呈现失效**：是否所有剧情都被写成同一种文字模式？例如高潮像摘要、日常像说明、压迫像口号、爽点没有反应链。
- [ ] **笔记本/反思滥用**：是否用"在笔记本上写几条总结"代替了通过场景和行动展示角色成长？如果每章结尾都有笔记本反思→P1

**发现P0的条件**：写作手法严重退化（大量"他决定了"/全靠抽象形容词/内心独白<20%/像作文不像小说）**或系统性退化**（整阶段文体突变/标志性短语泛滥/格调与设定脱节/空壳章节群/关键情绪兑现失败）
**发现P1的条件**：个别章节有写作手法问题但整体OK，或公式化倾向已出现但尚未失控

### Round D：物品/设定合理性

- [ ] 带入过去/异世界的物品符合时代合理性（没有方便面/电子设备/品牌包装等bug）
- [ ] 物价/工资/汇率精确且一致
- [ ] 称呼/用语符合年代
- [ ] 地理/交通/通讯方式符合时代

### Round D2：边界检查（新增——防止subagent越界）

- [ ] 本阶段最后一章的时间线是否在指定范围内？（没有超出）
- [ ] 本阶段是否写了结局/老死/时间跳跃到指定范围外？
- [ ] 本阶段是否提前写了后续阶段的内容？
- [ ] 如果发现越界→标记为P0→给出精确位置和修复方案；只有进入修复模式时，才删除越界部分并重写

### Round E：伏笔/预期状态追踪

- [ ] 本阶段计划回收的伏笔是否全部回收？
- [ ] 本阶段有没有新埋的伏笔？（更新foreshadow-tracker.md）
- [ ] 本阶段计划处理的预期是否全部处理？
- [ ] 伏笔有没有"遗忘超2个阶段"的？

### Round F：事实来源与设定同步

- [ ] 涉及真实年代、职业、制度、物价、交通、通讯、商品时，是否能在 era-database.md 或明确资料中找到依据？
- [ ] 如果章节与 era-database.md 冲突，是章节错了，还是设定库需要更新？必须二选一处理，不能留下双版本事实。
- [ ] 查到的事实、作者推断、为剧情做的保守取值是否分开标注？
- [ ] 如果不同来源互相矛盾，review中是否列出冲突，并说明采用哪个值作为后续写作标准？

**发现P0的条件**：事实错误会破坏核心机制或读者信任（如关键物价、政策、商品年代明显错误）
**发现P1的条件**：事实不够精确但不影响主线，需在下一阶段前补齐设定库

### Round G：角色真实性与差异化（新增）

- [ ] **角色是否脸谱化**：有没有"全员好人/全员睿智/全员大爱"的问题？人性的自私、矛盾、灰色地带是否体现了？
- [ ] **角色小传一致性**：每个角色的行为是否基于他的小传自然推理出来的？（不是作者安排他做什么，是基于他这个人自然会怎么做）
- [ ] **角色缺点体现**：每个角色至少2个人格缺点——在本阶段是否体现了？
- [ ] **角色私心体现**：每个角色有自己的小算盘——在本阶段是否体现了？
- [ ] **角色区分度**：不同角色说话的方式/思考的角度/行为的模式是否明显不同？还是所有人一个调子？
- [ ] **角色矛盾体现**：角色内心的矛盾冲突是否展示了？还是只有"善"没有"恶"？
- [ ] **角色账本同步**：character-ledger.md 是否记录了本阶段每个重要角色的当前处境、关系变化、最近关键选择、下一步最优选择？
- [ ] **关系压力变化**：重大事件后，角色关系是否发生了称呼、距离、信任、利益或权力结构的变化？还是下一章又回到原点？
- [ ] **台词遮名测试**：抽取每个重要角色最近3次出场，遮住名字后，能否凭台词、动作、判断角度分辨是谁？
- [ ] **非主角利益选择**：每个重要配角本阶段是否至少做过一次不完全服务主角利益的选择？

**发现P0的条件**：角色严重脸谱化（全员好人/没有人性的复杂面/所有角色一个模子）
**发现P1的条件**：个别角色有脸谱化倾向但整体OK；character-ledger.md 更新不完整但尚未造成剧情断裂

**角色漂移补充判定**：
- 如果两个以上重要角色互换台词后读者分不出，标为P1。
- 如果整阶段大多数角色都变成理性、善良、识大体的同一类人，标为P0。
- 如果角色经历重大事件后关系没有任何变化，且后续剧情仍按旧关系推进，标为P1；影响核心冲突时标为P0。

### Round H：冷启动追读与前30章钩子

如果review范围包含前三章，必须逐章检查：
- [ ] Ch1核心设定是否在场景中入场，并给主角带来即时信息、收益、优势、风险或麻烦？
- [ ] Ch1章末是否留下具体问题，而不是只留下氛围？
- [ ] Ch2是否展示限制、代价或外部压力，逼主角做选择？
- [ ] Ch3是否交付至少一个小爽点，且写出过程爽、见证者反应或状态变化？
- [ ] 三章后读者是否能说出“这本书后面会怎么爽”？

如果review范围包含Ch1-30，必须检查：
- [ ] 每章是否有短钩子：新问题、新风险、新收益、新反常信息、倒计时或对手动作。
- [ ] 每3-5章是否有中钩子：一个小单元问题被推进并最终兑换。
- [ ] 是否至少有一条长钩子持续推进：能力升级、隐藏敌人、身份风险、大地图、终局目标。
- [ ] 前30章末主角处境是否发生可见变化：钱、名声、资源、关系、身份风险、能力边界至少变化一项。
- [ ] 设定是否始终在行动/冲突/收益中出现，而不是传统文学式氛围化呈现。

**发现P0的条件**：前三章没有建立追读发动机；核心设定没有产生即时收益/冲突；前30章读完像短篇单元合集，看不出长线升级和后续期待。
**发现P1的条件**：有追读意识但钩子松散；前三章小爽点弱；前30章中钩子或长钩子推进不稳定。
**发现P2的条件**：个别章末钩子弱，可通过补尾钩或调整章节出口修复。

## P0/P1修复循环

review完成后，P0和P1必须修复（P2不阻塞）：

```
1. 按review报告修复P0和P1
2. 修复后做验证（检查修复是否引入新问题）
3. 如果验证发现新P0/P1 → 继续修复
4. 循环直到最新一轮P0/P1=0
5. 记录所有轮次在review.md中
```

**退出条件：最新一轮review显示P0=0且P1=0**

## 修复原则

- 精准patch，不改剧情结构
- P0优先修复（影响后续剧情的硬伤）
- 修复后必须验证（读修改后的文件确认修复正确）
- 如果修复引入新问题，新问题也要修
