# 定量风格分析技术参考

> 来源：Ch107-160 review 及 Ch1-280 全量 review 实战。这些技术发现了纯精读无法发现的 P0 级系统性问题。

工具名不是重点。示例里的 shell、`search_files` 或脚本都只是实现方式；当前环境没有某个工具时，用 `rg -n`、`rg -c`、`grep`、`awk` 或小脚本拿到同等统计即可。

---

## 技术零：破折号/碎片化密度分析（最高优先级）

### 为什么排第一

破折号密度是比短语频次更底层的退化探测器。短语频次（如"好家伙"次数）只能检测填充词泛滥；破折号密度直接测量句子结构的变化——从正常散文到"每个词之间加破折号"的碎片化断奏体。**它是唯一能推翻既有 review 核心判断的客观指标。**

### 命令

```bash
cd stories/<项目>/drafts && for f in chapter-*.md; do
  n="${f#chapter-}"; n="${n%.md}"; n=$((10#$n))
  b=$(wc -c < "$f"); d=$(grep -o "——" "$f" 2>/dev/null | wc -l | tr -d ' ')
  printf "%d\t%s\t%s\n" "$n" "$b" "$d"
done | awk -F'\t' '{
  bytes=$2; dashes=$3; chn=$1+0
  if (chn<=60) bin="act1     Ch1-60   "
  else if (chn<=106) bin="interim  Ch61-106"
  else if (chn<=160) bin="act3     Ch107-160"
  else bin="later    Ch161+   "
  sumB[bin]+=bytes; sumD[bin]+=dashes; n[bin]++
}
END {
  for (b in n) printf "%s  n=%-3d avgBytes=%-7.0f avgDashes/KB=%.2f\n", b, n[b], sumB[b]/n[b], sumD[b]*1024/sumB[b]
}'
```

### 判定标准

| 密度 | 判定 | 说明 |
|------|------|------|
| ≤10/KB | ✅ 正常散文 | 健康的叙事节奏 |
| 10-20/KB | 🟡 偏高 | 抒情/蒙太奇章节可接受；叙事章节需关注 |
| >20/KB | 🔴 碎片化断奏体 | P0 级文体退化。几乎每个词之间都有破折号 |
| >40/KB | 🔴🔴 极端碎片化 | 句子完全被拆碎，阅读体验从"小说"退化成"诗歌体大纲" |

### 实战案例（两界穿梭 Ch1-280 全量）

| 区间 | avg 破折号/KB | 判定 |
|------|--------------|------|
| Ch1-60 | 8.89 | ✅ 正常散文 |
| Ch61-106 | **35.24** | 🔴 **碎片化巅峰**（峰值 Ch106=57.7/KB）——此前从未被 review 覆盖 |
| Ch107-160 | 1.84 | ✅ 正常散文（全书最低） |
| Ch161-200 | 5.27 | ✅ 优秀散文 |
| Ch201-240 | 8.89 | ✅ 正常 |
| Ch241-280 | 12.83 | 🟡 抒情/蒙太奇（结局可接受） |

### 教训：定量数据推翻了既有 review

**背景**：act3-review.md 声称 Ch107-160 全部 54 章使用"超碎片化断奏体"，引用了 Ch107 开篇"林北——回到了——2025年"作为证据，并据此判定 P0。

**定量分析结果**：Ch107-160 实际平均破折号密度仅 1.84/KB（全书最低区间）。精读验证 Ch107 实际开篇为"林北睁开眼的时候，天花板上的水渍还在——左上角那块，像一片不对称的云"——优秀的正常散文。act3-review 引用的文本在当前文件中不存在。

**根因**：章节文件在 act3-review 之后被重写过，但 review 报告没有更新。review 引用的"证据"来自旧版本。

**结论**：既有 review 的结论可能基于历史版本。精读前必须用破折号密度等定量数据验证既有 review 的核心判断是否仍成立。

## 技术一：章节体量分析

### 命令

```bash
cd stories/<项目>/drafts && for i in $(seq NN MM); do f="chapter-${i}.md"; if [ -f "$f" ]; then echo "Ch$i: $(wc -l < "$f")L $(wc -c < "$f")B"; fi; done
```

### 实战案例（两界穿梭 Ch60-160）

发现 Ch141-147 区间体量偏短（**注：以下数据来自历史版本，全量 review 实测平均 5.5KB，可能已被扩写**）：

| 章节区间 | 平均字节 | 状态 |
|----------|---------|------|
| Ch60-106 | ~8.5KB | 正常 |
| Ch107-140 | ~11KB | 正常（篇幅偏长） |
| **Ch141-147** | **~4.5KB（历史）** | **曾异常缩水——全量 review 实测已恢复到 ~5.5KB** |
| Ch148-160 | ~8.5KB | 恢复正常 |

Ch143（马三鸡蛋生意做大）只有 3.3KB——实际内容用一个段落就能概括。Ch147（钱老来店观察）只有 3.8KB——信息量极低（"一个老头来看了看走了"）。

**处理建议**：体量异常的章节应考虑合并到相邻章节，或扩写到正常篇幅。

## 技术二：标志性短语频次分析

### 命令

```
search_files(
  pattern="好家伙",          # 主角口头禅，从 tone-guide 获取
  path="stories/<项目>/drafts",
  file_glob="chapter-1[0-5]*.md",  # 本阶段章节范围
  output_mode="count"
)
```

再对本阶段和上一阶段各跑一次，对比密度。

### 实战案例（两界穿梭）

| 章节区间 | "好家伙"总次数 | 平均每章 | 判定 |
|----------|--------------|---------|------|
| Ch40-59 | 4 | 0.2次 | 正常——标志性反应，有冲击力 |
| Ch100-160 | **468** | **8.7次** | **失控——退化成机械填充词** |

极端案例：Ch152/Ch156 各 20 次（约每 27 行一次）。

**判定标准**：标志性短语密度超过上一阶段 3 倍 → P0 级退化信号。

## 技术三：收尾公式化检测

### 命令

```
search_files(
  pattern="一步一步",        # 从已读章节中归纳出的高频收尾语
  path="stories/<项目>/drafts",
  file_glob="chapter-1[0-5]*.md",
  output_mode="count"
)
```

然后抽读最后几章的结尾 10 行，确认是否公式化。

### 实战案例

"一步一步"作为章节最后一行出现在 Ch107-160 至少 25 章中（占 46%）。加上几乎每章结尾都有"好家伙 → 合上笔记本 → 一步一步"三连收尾。

**判定标准**：相同收尾模式出现在 >30% 的章节中 → P1 级公式化。

## 技术四：跨阶段风格对比

### 方法

1. 精读本阶段 5-8 章关键章节
2. **回头读 2-3 章上一阶段的章节**（尤其是上次 review 赞扬过的章节）
3. 对比：句式节奏 / 内心独白呈现方式 / 信息密度 / 搞笑比重

### 实战案例（两界穿梭）

> ⚠️ **以下案例已修正（2026-07-01 全量 review）。** 原版案例错误地将 Ch107 引用为碎片化断奏体——实际 Ch107 是正常散文（破折号密度 1.84/KB，全书最低）。真正的碎片化巅峰在 Ch61-106（35.2/KB），是一个此前从未被 review 覆盖的盲区。详见技术零。

**Ch47（act1，正常散文）**：
```
春天的临江县。桃花开了。油菜花黄了一片。空气里弥漫着花粉和泥土的甜味。
```

**Ch106（interim，碎片化断奏体巅峰——破折号密度 57.7/KB）**：
```
孙主任——来了。
——
不是——来——买东西。
是——来——"看看"。
"看看"——他——儿子——到底——在——干什么。
```

→ 几乎每个词之间都插入了破折号。一句话拆成5-8行。大量单句成段。

**这不是偶发现象——这是 Ch61-106 共 46 章的一致风格。** 而下一区间 Ch107-160 已回归正常散文（1.84/KB）。跨阶段对比能发现这种突变——单独看一个区间时不易察觉，但定量数据（破折号密度）让差异无法隐藏。

**关键教训**：跨阶段对比 + 定量数据是发现系统性风格漂移的最可靠方法。必须在 Round C 中强制执行。且必须用定量数据验证既有 review 的结论是否仍成立。

## 检查清单（每次 review 的 Step 0 必跑）

- [ ] 跑破折号密度分析（`grep -o "——" | wc -l`），标记碎片化区间 —— **最高优先**
- [ ] 用定量数据验证既有 review 的核心判断是否仍成立（章节可能已重写）
- [ ] 跑章节体量分析（`wc -l` + `wc -c`），标记异常缩水章节
- [ ] 跑标志性短语频次（`search_files output_mode='count'`），本阶段 + 上阶段各一次
- [ ] 跑收尾语频次，检测公式化
- [ ] 精读前先读 2-3 章上一阶段章节建立风格基线
