# 批量重写协议：退化诊断与修复

> 本文件是 SKILL.md「批量重写」章节的详细参考。当用户要求"修正文风退化"或"重写Ch107-160"时阅读本文件。

工具名不是重点。表格里的 `search_files` 表示“对章节范围做关键词/计数扫描”；在 Codex 环境优先用 `rg -n`、`rg -c` 或小脚本等价完成。

---

## 一、退化诊断清单

在开始重写之前，先用以下清单诊断退化程度。退化通常是系统性的——不是一个指标出问题，而是多个指标同时崩溃。

| 退化模式 | 正常水平 | 退化水平 | 诊断方法 |
|---------|---------|---------|---------|
| **碎片化断奏体** | 破折号用于语句内补充说明或场景分隔 | 几乎每个词/短语之间插入"——" | 肉眼扫描：连续3段是否出现词间"——" |
| **"好家伙"密度** | 0.2-3次/章 | 8-20次/章 | `search_files` 对章节范围搜 `好家伙`，`output_mode=count` |
| **内心独白占比** | 30-50%，含"嘴上一套心里一套"反差 | 70-80%，纯喃喃自语 | 肉眼估算：内心独白段落 vs 对话/动作段落比例 |
| **搞笑元素** | 认知错位笑点（现代认知碰撞时代现实） | 归零——全程沉思/温情/紧张 | 检查是否有现代词汇被1979人困惑的场景 |
| **公式化收尾** | 多样化（场景收尾/对话收尾/悬念收尾） | 25+章以相同模式结尾 | 读取每章最后3行，比较模式 |
| **笔记本反思** | 偶尔出现（每3-5章1次） | 每章结尾都写笔记本总结 | `search_files` 搜 `笔记本|写了一行` |
| **章节字节量** | 6-12KB | 骤降到3-4KB（空壳章节） | `search_files` 用 `output_mode=count` 对空内容模式统计 |
| **信息密度** | 每段推进剧情或角色状态 | 大量空白行+破折号占满篇幅，实际内容1/3 | 读完一章后问自己：故事往前走了几步？ |

## 二、风格基准对比（正常 vs 退化）

### 正常散文（Ch1-60风格，重写目标）

```
春天的临江县。桃花开了。油菜花黄了一片。空气里弥漫着花粉和泥土的甜味。
```
→ 正常中文散文节奏。一句话有完整的主谓宾和修饰。

```
好家伙。他想着，她什么时候开始查打火机的？第一次见面就在看了？不是——
```
→ "好家伙"是遇到真正意外时的标志性反应，每次出现都有冲击力。

### 退化碎片化（Ch107-160风格，重写对象）

```
林北——回到了——2025年。
——
从1983年11月——到2025年12月。
隔着——四十二年。
但——只隔了——一个念头。
空间——收。眼前——白。
睁开——眼。
出租屋。
```
→ 几乎每个词之间都插入了破折号。一句话拆成5-8行。大量单句成段。

**关键认知**：碎片化的文字不是"风格选择"——它是文风崩溃。重写时不是"调整"这些段落，而是完全忽略它们，从剧情事件出发重新写。

## 三、重写工作流详解

### Step 1: 读取退化章节 — 只提取剧情

读退化章节时，大脑切换到"考古模式"：
- **提取什么**：发生了什么事件？谁说了什么关键台词？角色状态有什么变化？推进了哪些伏笔？
- **忽略什么**：句式、标点、段落划分、内心独白的表达方式——这些全部抛弃

示例：Ch143（空壳3.3KB）提取结果：
> 马三鸡蛋生意做大了。从200/天到500/天。3个村变6个村。雇了铁柱和小六。月收入超300。超过利民店分成。孙建设问马三赚多少，林北说电视维修以后一个月300不是问题。两人站在门口看马三骑车过去。

就这么几句话。剩下的全是碎片化填充。重写时从这几句话出发。

### Step 2: 写作 — 用基准风格从零重写

- 用 Ch1-3 / act-review赞扬章节的句式节奏
- 恢复正常段落（2-5句话一段，不是每句一段）
- 恢复正常破折号使用（仅用于语句内补充说明或场景分隔线）
- 恢复内心独白的"反差"质量（嘴上一套心里一套）
- 恢复认知错位搞笑（2025认知碰撞1984现实）
- "好家伙"只在真正意外的时刻使用（每章0-1次）

### Step 3: 自检

使用 SKILL.md 中的「重写专用自检」清单。

### Step 4: 覆盖原文件

用当前环境可用的文件写入方式覆盖原 chapter-{NN}.md 文件。

### 空壳章节扩写技巧

空壳章节（3-5KB）的剧情骨架通常正确——只是没有血肉。扩写时不是"注水"，而是补充原章节遗漏的叙事层次：

1. **场景氛围**：退化版本跳过环境描写。补回光线/温度/声音/气味等感官细节（铁律二）。如：办公室的阳光反射、巷子里的青苔、绿豆汤的冰糖甜味。
2. **决策拉锯**：退化版本只写"他决定了"。补回完整决策链——本能反应→内心拉锯→身体先于意识→决定后的反噬（铁律一）。如：拒绝钢材时先写诱惑（九千块=一年收入），再写恐惧（官倒=坐牢），最后写决断。
3. **角色观察**：退化版本把配角当工具人。补回主角对配角的观察和判断——对方的表情/语气/手部动作/潜台词。如：张明远擦眼镜的细节暴露了他在做重要决定。
4. **历史纵深**：主角来自未来，他知道这些事件的后续。补回主角的"上帝视角"思考——这件事在历史中意味着什么/会通向哪里。如：喷气织机代表的不是一台机器，是一个时代转型的缩影。
5. **认知错位搞笑**：退化版本搞笑归零。补回两个世界的认知碰撞——2025年的一块三什么都买不到，1985年的一块三能买四斤大米。

扩写后正常篇幅：6-10KB。不要超过12KB（那是另一种退化——注水）。

### 重写中的风格漂移（Rewrite Drift）

批量重写本身也会退化。长session（30+章）写到后半段时，注意力下降，好家伙密度和碎片化倾向会悄悄回升。

实测数据（Ch129-160重写session）：Ch129-140平均3-8次/章，Ch150-160回升到10-15次/章。

预防：每重写10章回头读一篇早期重写章节校准文感。每批结束跑 grep -c 扫描。如果session截断续接，先读最好的一章做基准。

修复：对漂移章节做二次精修——删机械填充的好家伙、合并碎片化段落、变换公式化收尾。不需要完全重写。

## 四、批量重写的Token管理

大范围重写（50+章）需要分批：

| 策略 | 说明 |
|------|------|
| 每批5-8章 | 避免单次输出token超限 |
| 并行读取 | 一次 `read_file` 读取2-3章退化原文（只看剧情），减少往返 |
| 进度报告 | 每批完成后报告已完成/总数 |
| 截断恢复 | 如果session被截断，最后一条消息应包含：已完成章节列表、下一章编号、当前状态摘要 |

### Session截断时的摘要模板

```
已完成：Ch107-128（22章重写）
未完成：Ch129-160（32章）
下一章：Ch129（沈父来访考试——已读头部，剧情已提取）
进度：22/54
```

## 五、验证命令

重写完成后（或每批完成后），运行以下验证：

### "好家伙"密度验证

```
search_files 对 chapter-1[0-6]*.md 搜 "好家伙"，output_mode=count
```
正常：每章0-2次（纯动作/对话章）；3-8次可接受（内心独白密集章/高潮章）。退化：10+次且机械重复（每次出现都没有新的冲击感）。

**注意**：不要过度压制"好家伙"——它是角色标志性口头禅，在认知错位/真正意外的时刻使用是正确的。重写目标是消除"机械填充式"的使用（退化版本用"好家伙"当段落分隔符），而非完全消灭。判断标准：如果删掉某处"好家伙"后读者不会觉得少了什么，那就是机械填充；如果删掉后该处的冲击力明显下降，那就是合理使用。

### 碎片化断奏体验证

```
search_files 对重写章节搜 "^.{1,3}——"（行首极短文本后跟破折号）
```
正常：极少命中。退化：大量命中。

### 元引用验证

```
search_files 对重写章节搜 "outline|concept|worldbuilding|设定|第.阶段|foreshadow"
```
命中即为违规（见 SKILL.md Pitfall 1）。

## 六、实战案例

### 案例1：《两界穿梭》Ch107-128（22章）

- 退化范围：从Ch107开始全部碎片化断奏体
- 退化指标：好家伙8.7次/章，Ch141-147平均4.5KB（空壳），25+章公式化收尾
- 重写方法：读Ch1-3+Ch47/Ch55基准，逐章提取剧情完全重写
- 重写后：碎片化归零，好家伙0-1次/章

### 案例2：《两界穿梭》Ch129-160（32章）

- 退化范围：与案例1连续，同样碎片化断奏体
- 退化指标：好家伙6-15次/章，空壳章节Ch141-147为3.3-6.1KB
- 重写方法：读Ch128（案例1已完成章）作为基准，逐章提取剧情完全重写
- 空壳扩写：Ch143从3.3KB扩写到5.3KB（马三鸡蛋生意），Ch147从3.8KB扩写到6.2KB（钱老来访）
- 重写后：好家伙Ch129-140控制3-8次/章（合理），Ch150-160回升10-15次/章（rewrite drift，需二次精修）
- 教训：32章单session过长，后半段出现rewrite drift。建议单session不超过20-25章，或每10章强制重读基准章节

### 案例3：《两界穿梭》Ch61-106（46章）

- 退化范围：渐进式碎片化，从Ch61（轻度）到Ch106（重度）
- 退化指标：
  - 好家伙密度：0-4次/章（正常范围——与案例1/2不同，好家伙不是主要问题）
  - 碎片化类型：**段落碎片化为主**（大量单句成段）+ **句内"——"过度使用**，而非词间"——"
  - 章节字节量：5.8-14.7KB（无空壳章节，与案例1/2不同）
  - 元引用：Ch103原文含"林北在Ch101说过"（章节自指——用Ch缩写形式，更难被grep抓到），Ch100原文含"（第二阶段·中）"阶段标签
- 重写方法：读Ch60+Ch107双重基准，逐章提取剧情完全重写
- 重写后：好家伙0-1次/章（46章共8处），碎片化归零，平均6.9KB，元引用0
- 教训：
  1. **碎片化断奏体有多种表现形态**——不总是"好家伙8-20次+3KB空壳章节"。有时好家伙密度正常、章节体积正常，但读感极差（单句成段+句内"——"泛滥）。诊断时必须**读样章肉眼感受读感**，不能只靠grep数字。
  2. **单session 46章重写是验证过的最大批量**——可行，但接近上限。重写比新写轻量（每章只需1次read + 1次write，无outline/architecture咨询），因此容量上限高于新写（25-35章）。46章重写约需100次工具调用。
  3. **双重基准校准法**：读退化范围**两侧**的好章节（前一阶段末尾Ch60 + 后一阶段开头Ch107），形成"夹击"基准。比只读一侧更稳定——两个基准的共性（正常段落长度、标点习惯）更可信。
  4. **"——"不可能归零也不应归零**——正常文学用法（对话引入、句内补充说明、场景分隔）应保留。重写目标是消除碎片化排版（单句成段、词间破折号），不是消灭破折号。验证时用 `.{1,4}——` 模式检查，命中后逐条判断是碎片化还是正常用法。
