# 写作经验路由协议

> 写作经验库来自对已有火爆小说的拆解。它不能只作为"可参考资料"挂在 prompt 里，而要变成章节执行约束：这一章要用哪类经验，解决哪个写作问题，写完如何验证。

## 一、经验库的使用原则

经验库不是用来增加术语的，而是把"会写小说"落实成动作：

- 场景如何进入，不靠说明。
- 冲突如何升级，不靠巧合。
- 对话如何有潜台词，不直说立场。
- 爽点如何有过程，不只给结果。
- 角色如何通过选择暴露性格，不靠旁白贴标签。

每次加载经验库后，必须提炼出本章的 3-5 条执行约束。不要把经验库原文复述到章节里。

## 二、先判情绪路由，再选经验库

经验库选择必须服务本章的情绪路由卡。不要只按题材标签选文件；同一个“商战”场景可能是压迫、博弈、爽点兑换或余波，文字策略不同。

| 呈现模式 | 优先经验文件 | 执行重点 |
| --- | --- | --- |
| 轻松搞笑 | writing-exp-1-1 / writing-exp-1-2 | 认知错位、反差、误判、极短句收尾 |
| 烟火气/温情 | writing-exp-1-4 | 物件、饭菜、声音、称呼、手上动作承载关系变化 |
| 严肃/压迫/危机 | writing-exp-1-3 | 选择收窄、沉默、代价、倒计时、身体反应 |
| 商战/博弈 | writing-exp-1-5 / writing-exp-4 | 底牌、假话、资源交换、社会规则伪装 |
| 爽点兑换 | writing-exp-1-6 / writing-exp-3 / writing-exp-3-2 | 铺垫回收、释放瞬间、见证者反应、余波 |
| 群像/关系变化 | writing-exp-2-1 / 2-2 / 2-4 | 不同目标、不同语言节奏、关系细节变化 |
| 反派/对手压迫 | writing-exp-2-3 / writing-exp-1-3 | 对手按最优选择行动，压迫来自利益和信息差 |

## 三、经验路由表

| 本章问题 | 优先经验文件 | 转化成的执行约束 |
| --- | --- | --- |
| 场景偏干、只有事件 | writing-exp-1-4 / writing-craft-summary | 每个关键场景至少有2种感官细节；用物、人、动作带出环境 |
| 搞笑不足或变尬 | writing-exp-1-1 / writing-exp-1-2 | 笑点来自认知错位；角色不犯蠢；笑点短句收尾 |
| 危机没压迫感 | writing-exp-1-3 | 先给限制，再缩小选择，再让角色付代价 |
| 爽点无过程 | writing-exp-1-6 | 爽点前必须有阻碍、误判、代价或反转 |
| 谈判/商战像说明文 | writing-exp-1-5 | 每个参与者都有底牌、假话和不能退让的线 |
| 群像混成一团 | writing-exp-2-1 | 同场角色必须有不同目标、不同语言节奏、不同误判 |
| 配角像工具人 | writing-exp-2-2 | 配角出场要推进自己的弧线，不只服务主角 |
| 反派降智 | writing-exp-2-3 | 反派按自己的利益做最优选择，失败来自信息差/代价/主角反制 |
| 女性角色扁平 | writing-exp-2-4 | 她有自己的目标、判断和代价，不只作为奖励或情绪支持 |
| 权力/认可写虚 | writing-exp-3-2 | 权力通过资源分配、称呼变化、行动权限表现 |
| 社会规律太直白 | writing-exp-4 | 规律藏在办事流程、人情交换、价格、称呼和风险里 |

## 四、章节开写前的经验摘要

写正文前先生成一个很短的经验摘要，供自己执行：

```markdown
本章经验路由：
- 情绪路由卡摘要：
- 本章主要风险：
- 已加载经验：
- 转化为执行约束：
  1.
  2.
  3.
- 写完验证：
```

这个摘要不要写入正文文件。它是写作前的作战笔记。

## 五、经验库缺失时的处理

如果路径中没有对应经验文件，不要假装已读取。说明缺失，并用本协议中的通用约束临时代替；同时在最终反馈中标记"经验库文件缺失，使用内置路由规则"。

## 六、review 如何检查经验是否真的被用上

不要检查"是否提到了经验库"，要检查经验是否改变了正文：

- 危机章有没有选择收窄和代价？
- 群像章是否每个人都有不同目标？
- 爽点章是否有过程，不只是结果到账？
- 目标体感是否兑现，还是只在大纲里写了情绪标签？
- 搞笑章是否来自认知错位，而不是角色犯蠢？
- 商战/谈判章是否每个人都有底牌和假话？

如果经验库被加载但正文仍是事件摘要，标为P1；如果整批章节都没有把经验转化为场景、对话、冲突和选择，标为P0。
