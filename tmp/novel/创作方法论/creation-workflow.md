# 小说创作完整流程

> 本文档是创作方法论的总执行手册。所有模型文档是各环节的深度参考，本文档把它们串成一条完整的、逻辑闭环的操作链。
> 每部新小说从第一行到交付，严格按此流程执行。

---

## 流程总览

```
Phase 0：构思（设定+设计）
  0-1 创意入口
  0-2 世界观构建
  0-3 腔调设定
  0-4 情绪呈现路由与措辞技法选择
  0-5 人物设定
  0-6 大纲与节奏规划
  0-7 前30章追读钩子链设计
  0-8 伏笔与预期清单初建
  0-9 前三章手动写作

Phase 1：初稿（逐章串行写作）
  1-1 逐章串行写作（含阶段性设定review）
  1-2 全量八维度review
  1-3 P0/P1修复循环到收敛

Phase 2：润色（不改内容只改阅读体验）

Phase 3：整理（合并+归集）
```

每个阶段有硬门禁——前一个阶段的所有检查项全部通过，才能进入下一个阶段。

**核心原则：章节数、阶段数、每章字数均由剧情自然生长决定，不预设任何数量约束。叙述声音保持稳定，具体文字呈现根据剧情功能动态调度。网络小说开局必须先满足追读：前三章建立发动机，前30章形成短/中/长钩子链。**

---

## Phase 0：构思

### 0-1 创意入口

**输入**：无
**产出**：concept.md
**使用模型**：创意入口生成器.md（v4）

操作步骤：
1. **确定代入感方向**（Step 0）：读者要体验什么人生？结果目标（1-2种：财富/权力/武力/超凡/认可/美人/自由）+过程体验（2-3种：赚钱快感/碾压爽感/逆袭解气/博弈紧张/成长蜕变/日常温情/探索新鲜/认知震撼/幽默调味）
2. **想画面**（Step 1）：闭眼想一个"一看就想追"的画面/场景/瞬间。不是从方向推导出概念，是从一个有趣的画面中长出创意。
3. **从画面提炼核心机制**（Step 2）：触发条件/使用规则/限制边界/升级路径
4. **有趣性检验**（Step 3）：看到这个机制，第一反应是"我想知道然后呢"吗？不通过直接淘汰。
5. **代入感+持续性+代入门槛三检验**（Step 4）
6. 从通过的方案中选5个最优
7. 择优1个
8. 回头贴标签（引擎/限制/场景——用于归类对齐，不用于生成）
9. 写入concept.md：核心机制完整规则+代入感定义（结果40%+过程60%）+格调定义（主角气质×环境×化学反应）+有趣性检验+三检验记录

**检查项**：
- [ ] 有趣性检验通过（看到机制第一反应是"我想知道然后呢"）
- [ ] 代入感方向明确（结果40%是什么？过程60%是什么？）
- [ ] 核心机制规则清晰（2-3句话说清）
- [ ] 核心机制有限制且限制能产生故事
- [ ] 核心机制有升级路径
- [ ] 核心机制有可无限复制的变量X
- [ ] 结果分阶段（多个阶段性目标递进）
- [ ] 过程体验多样化（2-3种交替）
- [ ] 格调的化学反应有独特性
- [ ] 代入门槛通过（主角=普通读者）
- [ ] 三检验全部通过
- [ ] concept.md包含前三章冷启动承诺和前30章追读钩子雏形

### 0-2 世界观构建

**输入**：concept.md
**产出**：worldbuilding.md
**使用模型**：worldbuilding-model.md

操作步骤：
1. **第零层：形而上学基础**
   - 主角核心机制：完整规则是什么？世界必须具备什么条件？
   - 固定维度审视：规则来源/生死结构/行为后果/世界边界/因果与认知
   - 追问链：从主角核心机制出发反复追问，直到不可再分
   - 向下传导：第零层的每个选择怎么影响六层结构
2. **六层社会结构**：物质基础→经济系统→权力结构→社会生态→文化价值观→主角能力交汇点
3. **五条二阶闭环**：经济/权力/阶层/价值观/主角冲击——每条追溯到第零层
4. 12个检验问题自检

**检查项**：
- [ ] 第零层的主角核心机制能解释主角独特性的来源
- [ ] 五个固定维度全部审视过
- [ ] 六层结构每一层至少有一处可追溯到第零层的影响
- [ ] 五条二阶闭环首尾相连
- [ ] 12个检验问题全部能回答

### 0-3 腔调设定

**输入**：concept.md、worldbuilding.md
**产出**：tone-guide.md
**使用模型**：tone-model.md；案例分析/tone-analysis-{1,2,3}.md 仅在文件存在时作为可选参考

操作步骤：
1. 定义九个维度：幽默发动机→幽默类型组合→笑点密度分布→叙述声音人格→语言质感→情感节奏→氛围底色→标志性手法→读者体感目标
2. 每个维度的选择必须能追溯到concept.md的核心机制和worldbuilding.md的世界观
3. 明确区分：全书稳定叙述声音是什么；不同剧情功能下，轻松/搞笑/烟火气/压迫/严肃/爽点兑换等情绪如何切换

**检查项**：
- [ ] 九个维度全部定义
- [ ] 主导幽默发动机跟核心机制匹配
- [ ] 2-4个标志性手法是这部小说独有的
- [ ] 氛围底色跟情感基调一致
- [ ] tone-guide没有把全书钉成单一情绪，而是定义了稳定声音和动态情绪边界

### 0-4 情绪呈现路由与措辞技法选择

**输入**：concept.md、worldbuilding.md、tone-guide.md
**产出**：writing-guide.md、emotion-style-map.md
**使用模型**：emotion-style-routing.md、style-model.md、scene-model.md、../写作经验库/index.md；案例分析/style-analysis-{1,2,3}.md 仅在文件存在时作为可选参考

操作步骤：
1. 读取emotion-style-routing.md，先建立“稳定叙述基准 + 动态情绪模式表”
2. 六层技法选择：措辞层→文风层→剧情层→设定呈现层→人物刻画层→社会运转呈现层
3. 每个选择必须能追溯到tone-guide.md的对应维度
4. 为本书高频剧情功能建立情绪路由：铺垫/压迫/发现/博弈/日常缓冲/爽点兑换/余波/转场分别怎么写
5. **查阅写作经验库**：根据本部小说的格调方向，从写作经验库中找到对应的专题（如搞笑查1-1、紧张查1-3、商战查1-5等），将创作指引纳入writing-guide.md
6. 在writing-guide.md中明确列出"本部小说写作时必须参考的经验库专题"

**检查项**：
- [ ] 感叹词谱系定义完整
- [ ] 基础句式节奏型确定
- [ ] 信息投放的主导方式确定
- [ ] 技法选择跟tone-guide一致
- [ ] emotion-style-map.md包含情绪路由卡模板、呈现模式表、章长策略
- [ ] 每种高频呈现模式都有“文字策略”和“禁止写法”
- [ ] writing-guide.md引用了写作经验库中至少2份相关专题

### 0-5 人物设定

**输入**：concept.md、worldbuilding.md
**产出**：characters.md
**使用模型**：../写作经验库/专题/writing-exp-2-1.md（群像编织）、2-2.md（配角弧线）、2-3.md（反派塑造）、2-4.md（女性角色）

操作步骤：
1. 设计主角：基本信息/核心设定/性格/获得机制前的生活/成长弧线
2. 设计3-5个核心配角：每个配角有自己的目标/动机/秘密/弧线
3. 每个配角参照写作经验库的三重定锚公式（功能锚点+反差动作+标志物）
4. 反派参照写作经验库的反派塑造经验（反派的重量来自他的"对"不来自他的"坏"）

**检查项**：
- [ ] 主角的成长弧线清晰
- [ ] 每个配角有自己的目标（不是纯工具人）
- [ ] 反派有合理性
- [ ] 女性角色有主体性
- [ ] 配角的出场方式参照了三重定锚公式

### 0-6 大纲与节奏规划

**输入**：以上所有设定文件
**产出**：outline.md
**使用模型**：pacing-model.md、payoff-model.md

操作步骤：
1. **按剧情阶段划分大纲**（不预设章节数，不预设阶段数）。每个阶段定义：核心问题/主要事件/主角弧线推进/阶段高潮。阶段数量由故事需要决定。
2. 用pacing-model检查宏观节奏（冷启动期→发展期→深水期→高潮期→余韵——用剧情特征描述阶段，不用章节区间）：
   - 冷启动期：核心机制必须尽快激活，前几章至少1个小爽点
   - 发展期：扩展世界，每几章有小爽点
   - 深水期：维度升级，至少1次反转
   - 高潮期：多线交汇，终极爽点
   - 余韵：放慢节奏收束
3. 用payoff-model标注爽点位置和铺垫-释放周期
4. 用emotion-style-routing.md给关键节点标注完整情绪路由：剧情功能、读者期待、目标体感、呈现模式、爽点阶段和章长策略
5. 用webnovel-opening-hook-model.md检查前30章的冷启动节奏和追读牵引
6. 画节奏曲线图（文字版）

**检查项**：
- [ ] 大纲按剧情阶段划分完成，每个阶段有明确的核心事件
- [ ] 冷启动期前几章有爽点
- [ ] 爽点间距合理（不连续多章无爽点）
- [ ] 大高潮前有足够的铺垫
- [ ] 节奏曲线图无连续空白区
- [ ] outline节点不是只有“情绪标记”，而是包含剧情功能、读者期待、目标体感、呈现模式、爽点阶段和章长策略
- [ ] 前三章满足“设定入场→限制加压→小爽点兑换”
- [ ] 前30章有短钩子/中钩子/长钩子链

### 0-7 前30章追读钩子链设计

**输入**：concept.md、outline.md、emotion-style-map.md
**产出**：front-30-hook-map.md
**使用模型**：webnovel-opening-hook-model.md、payoff-model.md、expectation-model.md

操作步骤：
1. 读取webnovel-opening-hook-model.md，明确前三章冷启动职责。
2. 为Ch1-3分别标注：设定入场方式、限制/代价、小爽点、章末钩子。
3. 为前30章设计短钩子、中钩子、长钩子：每章有章末牵引，每3-5章有单元兑现，至少1条长线期待持续推进。
4. 检查设定是否在行动/冲突/收益中出现，而不是只做氛围或说明。
5. 写入front-30-hook-map.md。

**检查项**：
- [ ] front-30-hook-map.md已生成
- [ ] Ch1-3冷启动职责明确
- [ ] 前30章每章有短钩子
- [ ] 前30章每3-5章有中钩子兑现
- [ ] 至少1条长线钩子持续推进
- [ ] 前30章末能形成“这本书后面会怎么爽”的整体印象

### 0-8 伏笔与预期清单初建

**输入**：outline.md、worldbuilding.md
**产出**：foreshadow-tracker.md（伏笔追踪表）、expectation-list.md（预期清单）
**使用模型**：foreshadow-model.md、expectation-model.md

操作步骤：
1. 从worldbuilding的五条二阶闭环中提取长线伏笔种子，登记入伏笔追踪表
2. 从outline中提取每个阶段计划的中/短线伏笔，登记入表
3. 建立预期清单：能力预期/情感预期/悬念预期/命运预期，每条标注"满足/延迟/颠覆"策略和计划时机

**检查项**：
- [ ] 伏笔追踪表至少有5条长线伏笔（来自二阶闭环）
- [ ] 每个阶段至少有1条中线伏笔
- [ ] 预期清单覆盖四类预期
- [ ] 预期清单与伏笔追踪表交叉验证

### 0-9 前三章手动写作

**输入**：以上所有设定文件
**产出**：drafts/chapter-01.md ~ chapter-03.md
**使用模型**：scene-model.md、../写作经验库/index.md

操作步骤：
1. 手动写前三章，建立全书的叙述声音
2. 前三章必须完成：核心机制激活、主角核心魅力展现、世界观初步展示、至少1个小爽点、核心配角的首次出场
3. 按front-30-hook-map.md执行冷启动：Ch1设定入场并产生即时反应；Ch2限制/代价卡住主角；Ch3交付第一个小爽点并打开更大期待
4. 每章开写前先生成情绪路由卡，明确这一章要给读者什么体感、用什么文字模式兑现
5. 每章写完后用scene-model检查场景入口出口、场景类型配比
6. **查阅写作经验库**：写作时参考对应的创作指引（决策过程完整展示/氛围靠感官细节/内心独白30-50%/对话叙述配合等）

**检查项**：
- [ ] 前三章的叙述声音跟tone-guide.md一致
- [ ] 前三章激活了核心机制
- [ ] 第一章有至少1个小爽点
- [ ] 前三章结尾留了钩子
- [ ] 前三章让读者看出“这本书后面会怎么爽”
- [ ] 设定在行动/冲突/收益中呈现，而不是传统文学式氛围化呈现
- [ ] 前三章分别完成了各自情绪路由卡的目标体感
- [ ] 决策过程完整展示（不是"他决定了"而是完整思考链）
- [ ] 氛围靠感官细节（不是抽象形容词）
- [ ] 内心独白占比足够（30-50%）

## Phase 1：初稿

### 1-1 逐章串行写作

**输入**：Phase 0全部产出物
**产出**：drafts/chapter-04.md ~ chapter-XX.md（XX由剧情自然生长决定）
**使用模型**：全部模型 + ../写作经验库

操作步骤：
1. 从第四章开始，严格串行——一章一章写，不并行
2. 每章写作前：读前一章确认衔接，检查本章在节奏曲线的什么位置，并生成情绪路由卡
3. 每章写作中：用scene-model检查场景构建，用payoff-model检查爽点周期阶段，用emotion-style-routing.md检查文字呈现是否匹配剧情功能
4. 如果本章在Ch1-30，读取front-30-hook-map.md，确认本章短钩子/中钩子/长钩子和设定入场方式
5. **每章写作中：查阅写作经验库中对应的创作指引**
6. 每章写作后：更新伏笔追踪表，检查技法一致性、情绪兑现和追读牵引
7. **每章字数由该章剧情需要决定**——不设上限下限

**阶段性设定review**：
每个剧情阶段完成后（章节数由剧情决定），停下来做设定review：
1. 读本阶段全部章节，对照设定文件检查一致性
2. 检查伏笔追踪表和预期清单
3. 检查节奏曲线
4. 检查情绪路由兑现：该爽的是否爽、该压的是否压、该松的是否有呼吸、该烟火气的是否有生活质感
5. **检查写作手法质量（新增）**：决策过程是否完整展示？氛围是否有感官细节？内心独白占比够吗？像小说还是像作文？参照写作经验库标准
6. 按需更新设定文件

**检查项**（每章）：
- [ ] 衔接前一章（无断裂）
- [ ] 场景入口出口有效
- [ ] 决策过程完整展示（不是"他决定了"）
- [ ] 氛围靠感官细节（不是抽象形容词）
- [ ] 内心独白占比足够（30-50%）
- [ ] 标志性手法出现（至少1次）
- [ ] 本章目标体感已兑现，文字呈现模式跟剧情功能匹配
- [ ] 若本章在Ch1-30，本章短钩子/中钩子/长钩子已兑现
- [ ] 设定在行动/冲突/收益中出现，而不是只做说明或氛围
- [ ] 无元引用
- [ ] 伏笔追踪表已更新

**检查项**（每阶段）：
- [ ] 设定一致性检查通过
- [ ] 伏笔追踪表状态更新
- [ ] 预期清单状态更新
- [ ] 节奏曲线符合规划
- [ ] 情绪模式分布合理，没有连续错用同一种文字模式
- [ ] 若阶段涉及Ch1-30，front-30-hook-map.md的追读链没有断裂
- [ ] **写作手法质量检查通过（新增）**

### 1-2 全量八维度review

**输入**：全部章节完成
**产出**：review.md
**使用模型**：`../novel-stage-review/SKILL.md` + 全部模型 + ../写作经验库/通用总结/writing-craft-summary.md

操作步骤：
1. 逐章通读全部正文
2. 按 `novel-stage-review` 做八维度审查：机制、人物、写作手法、物品/边界、伏笔/预期、事实来源、角色差异化、冷启动追读
3. 如果审查范围涉及Ch1-30，必须读取front-30-hook-map.md，逐章检查设定入场、即时收益/冲突、章末短钩子、中钩子、长钩子
4. 按P0/P1/P2分级，写入review.md
5. 用expectation-list验证全部预期已处理

**检查项**：
