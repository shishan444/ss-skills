# 小说润色与整理

创作流程的最后一步。输入是初稿全部完成的章节，输出是合并好的最终文件+整理好的目录。

## 职责边界

这是最后一道阅读体验整理，不是补救结构问题的工具。只要还有P0/P1，先回到 `novel-stage-review` 修；只要需要重写章节，回到 `novel-chapter-write`。润色不能掩盖剧情断裂、人物崩坏、时代错误或文风退化。

## 输入

- 全部章节初稿（drafts/chapter-{NN}.md）
- review已完成，P0/P1=0
- tone-guide.md（文风标准）

如果缺少 review 或 review 仍有P0/P1，不要进入润色。先输出阻塞原因和应回到哪个 skill。

## 输出

- 合并后的完整小说文件（{书名}.md）
- 整理好的目录结构
- 润色记录

## 执行步骤

> **批量处理脚本**：引号统一 + 填充词清除 + 标点标准化 + 章节合并可用 `scripts/polish_pipeline.py` 一键完成。运行后务必做 xxd 字节验证（见 1.2）。

### 第一步：润色（不改内容只改阅读体验）

**绝对禁止**：改剧情/改角色/改结构/改章节顺序/增删段落

#### 1.1 冗余表达清除

搜索并清除填充词：
- "也就是说" / "换句话说" / "简单来说" / "简而言之" / "说白了" / "总而言之"
- 过度的"其实" / "事实上" / "实际上"（保留角色自然口语中的合理使用）

清除方式：先搜索定位全部命中，写 Python 脚本批量清除（区分对话/旁白，见下方陷阱），记录清除数量。

**陷阱：对话检测的边界情况**
"行首是中文开引号 = 对话行"这个启发式在遇到 `旁白叙述+"对话内容"` 这种混合行时会误判——整行不以 `"` 开头，但实际包含对话，脚本会把对话里的填充词也删掉。
- 例：`林北想了想。"……也就是说，对空间的'认知'——不是老张的。"` ← 脚本会误删对话中的"也就是说"
- 规则：脚本清除后，必须 grep 残留填充词逐条人工复核。凡是出现在对话引号内的，恢复原文。

#### 1.2 引号统一

将所有英文直引号 `"` 转为中文弯引号 `\u201c/\u201d`。

注意事项：
- 先清理可能存在的反斜杠转义残留（`\"`，某次 subagent 写入可能批量引入数千处）
- 先清理尾部多余引号（连续两个 " 在行尾的）
- 然后做交替转换（奇数=开引号，偶数=闭引号）
- 用 Python 脚本批量处理，不要手动一章一章改

**严重陷阱：写文件工具可能吞掉 Unicode 弯引号**
有些写入工具会把脚本源码里的字面弯引号规范化成 ASCII `"`。脚本照常运行、报告"转换了 N 处"，但输出文件里的引号仍然是 ASCII——这是静默失败。

解决：脚本里用 Unicode 转义 `OPEN_Q = '\u201c'`、`CLOSE_Q = '\u201d'`，不直接写字面弯引号字符。

**验证（必须做）**
转换后用 xxd 检查字节，确认 `"` 变成了 `e2 80 9c`（不是 `22`）：
```bash
sed -n '11p' chapter-01.md | xxd | head -3
```
不要只靠 grep 计数判断——脚本被吞掉时 grep 结果有误导性。

#### 1.3 破折号检查

扫描含 5+ 个 `——` 的段落（用 Python 脚本按空行分段统计）。审查每一段是否过度——但注意以下**合法高密度场景**不算过度：
- 清单/报价段落（货物名——单价，连续列举）
- 快节奏对话中的戏剧性停顿（角色一口气报数、长辈说教的节奏感）
- tone-guide 定义的风格化短句节奏

只有当破折号用于替代本该是逗号/句号的普通叙述连接、且确实读起来割裂时，才改为标点。实测中 85/85 高密度段落均为合法风格，零修改。

#### 1.4 文风一致性

检查不同章节之间是否有明显的文风差异。如果有，统一到tone-guide.md的标准。

重点检查：
- 后期章节是否跟前三章的叙述声音一致
- 不同subagent写的章节之间是否有割裂感
- 感叹词谱系是否一致

#### 1.5 标点统一

- 省略号统一为"……"（六个点）
- 破折号统一为"——"（两个em-dash）
- 对话标点统一

### 第二步：整理

#### 2.1 章节合并

将全部章节合并为一个完整文件：

```
{书名}.md
├── 第一章 标题
│   （正文）
├── 第二章 标题
│   （正文）
├── ...
└── 最后一章 标题
    （正文）
```

章节之间用分隔线（`---`）隔开。合并时将章节标题从 `#` 降级为 `##`，书名用 `#`，保持 markdown 层级。注意章节内部可能也有 `---` 场景分隔符（合法，保留）。

#### 2.2 目录归集

整理目录结构：

```
stories/{序号}-{书名}/
├── {书名}.md          # 完整合并文件
├── concept.md         # 创意设定
├── worldbuilding.md   # 世界观
├── tone-guide.md      # 腔调
├── writing-guide.md   # 写作指南
├── outline.md         # 大纲
├── characters.md      # 人物
├── character-ledger.md # 角色状态账本
├── review.md          # review记录
├── foreshadow-tracker.md
├── expectation-list.md
├── act{N}-review.md   # 各阶段review
└── drafts/            # 章节草稿（保留）
```

#### 2.3 清理临时文件

删除写作过程中产生的临时文件：
- 辅助脚本（_*.py）
- 备份目录（*.backup-*）
- 诊断文件（_diag_*）

#### 2.4 润色记录

生成润色记录文件，记录：
- 清除了多少填充词
- 转换了多少引号
- 修改了多少处破折号
- 文风统一了多少处
- 总字数（润色前/润色后）

## 检查项

- [ ] 冗余表达清零（旁白清除 + 对话保留，人工复核残留）
- [ ] 引号全部统一为中文弯引号（xxd 字节验证通过，非 grep 计数）
- [ ] 破折号使用合理（高密度段落逐条审查，合法风格保留）
- [ ] 文风一致性通过
- [ ] 标点统一
- [ ] 无内容改动（只改阅读体验）
- [ ] 合并文件生成（`##` 章节标题层级，`---` 分隔）
- [ ] 目录结构统一
- [ ] 临时文件清理
- [ ] 润色记录生成
