# 剧情架构质量门槛与时代设定库

> 构建新小说架构、做大范围推演或修正设定时阅读。主 SKILL.md 只保留入口，详细标准在这里执行。

## 质量检查标准

### 引擎质量

- [ ] 故事引擎能支撑目标长度，不会中途熄火。
- [ ] 引擎熄火后有第二引擎接力。
- [ ] 引擎能产出足够多的自然剧情，不只是一两个高概念场景。

### 设定质量

- [ ] 每个设定都标注了"删掉会怎样"。
- [ ] 承重设定足够多，删掉就会让剧情崩。
- [ ] 没有空转设定：有设定但从不产出剧情。

### 映射质量

- [ ] 每条剧情都能追溯到支撑它的设定。
- [ ] 没有悬空剧情。
- [ ] 每个承重设定至少支撑多条剧情，而不是只服务一次。

### 压力质量

- [ ] 每个阶段的压力源不同，不能一直靠同一个威胁。
- [ ] 压力有接力：前一个压力衰减后新压力接上。
- [ ] 压力会逼出角色变化，不只是让主角过关。

### 角色质量

- [ ] 每个角色的最优选择都可能与主角利益发生冲突。
- [ ] 没有角色为了剧情方便被操控。
- [ ] 反派和对手智商在线。
- [ ] 每个角色有至少2个人格缺点和自己的私心。
- [ ] 角色之间的说话方式、思考角度、行为模式明确不同。
- [ ] 没有"全员好人/全员睿智"的问题。
- [ ] characters.md 已写稳定内核，character-ledger.md 已写当前状态账本。
- [ ] 每条重要关系边都标注了隐藏压力和下一次必然摩擦。
- [ ] 新增角色通过准入五问，不是为了跑腿、解释信息或临时制造情绪而出现。

### 时间线质量

- [ ] 历史事件按真实时间线发生。
- [ ] 设定状态变更追踪完整。
- [ ] 没有时间线冲突。

## 时代调研与设定库

时代调研在设定构建之后、写作之前执行。原因是：调研什么取决于设定了什么。先确定"1979年临江县+两界倒卖+棉纺厂"，才知道要查1979年的物价、棉纺厂体制和临江县社会背景。

### 调研内容

1. **历史物价调研**（涉及年代文/穿越时必须做）
   - 主角所在年代的工资水平。
   - 核心物资价格：粮、油、肉、布、自行车、手表、缝纫机、烟酒、日用品等。
   - 黑市价格 vs 计划价格。
   - 通胀前后的价格变化趋势。

2. **社会背景调研**
   - 票证、介绍信、户籍、单位制等制度如何运作。
   - 通讯方式：写信、电报、公用电话，以及明确没有什么。
   - 交通方式：火车、汽车、自行车的速度和价格。
   - 日常生活细节：吃穿住、娱乐、重要历史事件时间线。

3. **现代线调研**（涉及双穿/重生时必须做）
   - 主角在现代的真实生活方式：手机、网购、外卖、微信、短视频。
   - 主角职业/学校/社交圈的真实状态。
   - 现代物价：古董、收藏品、二手物品等。
   - 现代线中主角会使用的具体平台：淘宝、闲鱼、微信、抖音、美团等。

4. **物品合理性验证清单**
   - 每个主角会带过去的东西，验证那个年代是否存在或是否合理。
   - 每个倒卖商品，验证真实价格落差，不能靠想象编价格。

### era-database.md 要求

era-database.md 是后续所有写作的事实基础。物价、商品、制度、社会细节都从这里查。

每条关键事实要标明来源或证据类型；没有查到但基于常识补出的内容必须标为"推断"，不能伪装成事实。若不同来源冲突，把冲突列出来，并选择对小说更安全的保守值。

调研完成后，回头检查 worldbuilding.md、characters.md、outline.md。若调研发现设定中有不合理的物价、商品或社会细节，立即修正。
