# 超大范围推演实战格式参考

> 基于《两界穿梭》Ch161-280推演（68个节点/3个子阶段/120章/2.5故事年）实战提炼。

---

## 一、什么时候需要超大范围推演

- 用户要求"产出Ch161-280的剧情节点"（120章范围）
- 一个完整阶段（如第五阶段1985-1987）需要一次性推演完成
- 跨阶段推演（从一个阶段末到下一个阶段末）

标准范围推演产出20-50个节点。超大范围推演产出60-80个节点。

---

## 二、子阶段拆分方法

**核心原则：不要一口气推68个节点——拆成3个有独立弧线的子阶段。**

### 拆分维度

每个子阶段按以下维度独立设计：

1. **时间边界**：明确标注覆盖的时间范围（如"1985秋-冬"/"1986全年"/"1987全年"）
2. **核心主题**：用一个短语概括该子阶段的本质（如"暗流收紧"/"裂变"/"风暴与重生"）
3. **情绪弧线**：每个子阶段有独立的情绪起伏（不能三个子阶段都是"紧张→爽→紧张→爽"的重复模式）
4. **章节数估算**：约30-40章/子阶段

### 实战示例（《两界穿梭》Ch161-280）

```
子阶段A "暗流收紧"（Ch161-200, 1985秋-1985冬, N52-N71, 20节点）
  情绪弧线：温情→紧张→爽（翻盘）→悬念→温暖→满足
  核心：钱老科学检测/蝴蝶效应升级/老张-空间传承发现

子阶段B "裂变"（Ch201-240, 1986春-1986冬, N72-N94, 23节点）
  情绪弧线：紧张+爽→冲突→震撼→温暖→紧张→温暖
  核心：通胀冲击/马三分家/省城研究员/赵红梅系统分析

子阶段C "风暴与重生"（Ch241-280, 1987春-1987冬, N95-N119, 25节点）
  情绪弧线：紧张→低谷→大翻盘→震撼→温暖→恐惧→救赎
  核心：大低谷→转型/F001核心回收/感情定调/存在主义危机与救赎
```

### 子阶段之间的过渡设计

- 子阶段A结尾：阶段性收束节点（年终盘点/暂时过关）+ 悬念钩子（老张照片→空间传承）
- 子阶段B开头：用A的钩子引入新冲突（老张照片的发现推动B的传承线）
- 子阶段B结尾：多个角色的状态变化收束（分家/赵红梅军师/沈知秋知道秘密）
- 子阶段C开头：外部大事件打破B的收束（通胀加剧/税务查账）

## 三、Review文件驱动的节点设计

### 问题：review中的质量警告如何影响推演

act3-review.md中标记了三个P0问题：
1. 超碎片化断奏体（写作风格退化）
2. 好家伙泛滥（8.7次/章 vs Ch1-60的0.2次/章）
3. 搞笑元素归零（与concept.md设定"搞笑+奋斗"严重脱节）

### 这些P0在推演产出中的体现

1. **architecture-status.md的断裂点**：将写作质量问题列为🔴P0断裂点（与设定层面的断裂点并列）。标注修复方案和执行时机。

2. **推演质量自检**：在标准8项自检之外，新增：
   - [ ] 情绪路由是否考虑了review中的质量警告（如搞笑不足→增加"轻松搞笑/爽点兑换"呈现模式，并标出目标体感）
   - [ ] 至少一次真正的挫折→翻盘（review说翻盘爽感偏弱）

3. **节点情绪路由设计**：有意在推演节点中加入目标体感和呈现模式，例如"目标体感=翻盘爽，呈现模式=爽点兑换，章长策略=完整展开"；"目标体感=松一口气，呈现模式=烟火气/温情，章长策略=中等推进"。不能只写"爽/温暖/紧张"标签。

## 四、多线交叉节点的识别

超大范围推演中，多线交叉节点（2+条压力线交汇）是最精彩的剧情爆发点。

### 识别方法

在节点表的"压力线"列中标注该节点由哪几条线逼出。统计多线交叉节点数量：

```
单线节点：压力线列只有1个数字（如"4"=纯历史事件线）
双线节点：压力线列有2个数字（如"2,3"=角色利益+后果链）
多线交叉节点：压力线列有3+个数字（如"1,2,6"=设定+角色+伏笔）
```

### 质量标准

- 超大范围推演的多线交叉节点应≥10个
- 最精彩的节点通常是3条线以上的交叉（如设定+角色+伏笔回收同时交汇）

### 实战示例

```
N66 "老张说1953年捡到热的石头" 
  压力线：1,6（设定压力+伏笔回收）
  → 这是F001（空间上一个主人）的核心推进点

N98 "空间带来的货没有发票→税务漏洞暴露"
  压力线：1,3（设定压力+后果链）
  → 空间秘密首次在经济层面产生实际危机

N103 "大低谷：两界同时收紧"
  压力线：2,5（角色利益+情绪曲线）
  → 中期最大挫折点
```

## 五、伏笔管理的超大范围视角

超大范围推演（120章）中，伏笔回收/推进是核心节奏控制工具。

### 回收节奏

- **子阶段A**：推进3-5个伏笔（不回收大的，只推进）
- **子阶段B**：推进5-8个伏笔 + 回收1-2个中型的
- **子阶段C**：回收1-2个大型伏笔（全书级别的核心悬念）+ 推进终极伏笔

### 实战示例

```
子阶段A回收/推进：
  F015钱老线→部分回收（质检员）
  F009蝴蝶效应→深化推进
  F001空间传承→重大推进（老张照片/1953年热石头）

子阶段B回收/推进：
  F005马三线→部分回收（分家）
  F014赵红梅线→部分回收（技术军师）
  F013孙主任线→部分回收（归队）

子阶段C回收/推进：
  F001空间上一个主人→✅核心回收（N108老张确认）
  F010底层规律→高潮（N113沈父深谈）
  F009蝴蝶效应→深化至存在主义危机（N116-118）
```

## 六、architecture-status.md的更新范围

超大范围推演后，architecture-status.md需要全面重写（不是追加），因为：

1. 故事引擎可能新增/变化（如蝴蝶效应从辅助变主引擎）
2. 设定资产清单大量更新（每个设定推演后状态都变了）
3. 角色状态表需要覆盖推演后的状态（而非推演前）
4. 伏笔状态需要从"计划"更新为"推演后状态"
5. 断裂点需要包含review文件中的写作质量问题

### 关键：状态表要同时显示"推演前"和"推演后"

```
| 角色 | Ch160状态 | Ch280推演后状态 | 核心利益线 | 下一步最优选择 |
```

这样写作者和后续推演者都能看到变化轨迹。
