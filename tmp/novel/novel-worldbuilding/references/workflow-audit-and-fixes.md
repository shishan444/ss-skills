# 小说创作流程审计与修复清单

> 这个文件记录 novel skill 系统的流程级问题和修复方案。适用于重构流程、长篇中期盘点、阶段复盘后发现系统性退化时阅读。

## 问题一：角色构造只停留在设定表，缺少长篇状态账本

表现：
- characters.md 里有利益线、缺点、弧线，但写到中后期角色仍然同质化。
- 配角越来越像工具人，只负责递信息、帮主角、夸主角。
- 重大事件发生后，人物关系没有实质变化，下一章又回到原来的称呼、距离和信任程度。

根因：
- 稳定角色内核和当前状态混在一起，后续写作不知道哪些不能改、哪些必须更新。
- 没有记录角色最近关键选择，导致角色行为无法从过往经历自然推出来。
- 关系被写成"朋友/敌人/亲人"，没有写成持续产剧情的压力边。

修复：
- 新增 `character-ledger.md`，按阶段更新当前处境、关系变化、利益变化、最近关键选择、下一步最优选择、偏离警报。
- characters.md 保留稳定内核：一句话钉子、欲望恐惧、人格矛盾、行为算法、语言动作识别、关系压力。
- chapter-write 每章开写前做角色取样；stage-review 抽查最近3次出场做台词遮名测试。

## 问题二：写作经验库被当成参考资料，没有变成执行约束

表现：
- prompt 写了"参考写作经验库"，但正文仍然像事件摘要。
- 危机没有选择收窄，爽点没有过程，群像没有多目标冲突，对话没有潜台词。
- review 只检查文风退化，没有检查经验是否真的落到场景、冲突、对话和选择里。

根因：
- 缺少经验路由：不知道某类章节应该读哪类经验。
- 缺少转化步骤：读完经验后没有提炼为本章3-5条执行约束。
- 缺少验收：写完后没有检查经验是否改变了正文。

修复：
- 新增 `craft-experience-routing.md`，把章节问题映射到经验文件和执行约束。
- chapter-write 开写前生成经验摘要：本章主要风险、已加载经验、执行约束、写完验证。
- stage-review 增加"写作经验已转化"检查项。

## 问题三：worldbuilding、writing、review 之间有断点

表现：
- worldbuilding 产出人物设定，但 chapter-write 不一定读取。
- chapter-write 写完角色变化，但不一定回写角色状态。
- review 发现问题，但下一轮推演不一定吸收。

根因：
- 设定、执行、审计三环没有共同维护同一批状态文件。
- architecture-status.md 偏重故事引擎和设定状态，角色关系的细粒度变化不足。

修复：
- 统一状态文件：architecture-status.md 管故事引擎和设定压力，character-ledger.md 管人物与关系，foreshadow-tracker.md 管伏笔。
- 每5-8章小批次检查时更新 character-ledger.md。
- worldbuilding 推演必须读取最新 review 和 character-ledger.md。

## 问题四：长篇中期容易剧情合理、人物却变平

表现：
- 剧情节点都能找到设定支撑，但人物读起来越来越像同一种声音。
- 所有人都聪明、善良、识大体，冲突只来自外部事件。

根因：
- 压力测试偏剧情合理性，对人物活性和语言识别度约束不足。
- "角色智商在线"被误解为"角色不能犯错、不能自私、不能狭隘"。

修复：
- 角色质量不再只检查智商在线，还要检查私心、误判、盲区、羞耻点和不完全服务主角利益的选择。
- 关系边必须写"下一次必然摩擦"。
- stage-review 将全员同质化列为P0/P1门槛。

## 问题五：流程缺少回写动作

表现：
- 写作过程中产生新伏笔、新关系、新角色变化，但状态文件仍停留在旧版本。
- 下个 session 只读旧设定，导致重复、偏离或关系回滚。

根因：
- 过去流程强调"读上下文"，但没有强调"写完后回写状态"。

修复：
- chapter-write 批量写作每5-8章更新角色账本和伏笔状态。
- stage-review 必须输出 character-ledger.md / foreshadow-tracker.md 更新状态。
- worldbuilding 中期盘点以当前章节实态为准，不以旧 outline 映射为准。
